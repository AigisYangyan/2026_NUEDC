## 0. 一句话答案：哪边是左

> **灰度：G1（厂商丝印 P1，即与 14P 排线第 1 脚导通的那一路探头）装在车前进方向的最左侧。G1 = 左。**
>
> **电机/编码器：哪个 6P 插座是左轮，需要你们拍板（见 §3，固件命名当前是交叉的，推荐定义 U33 = 左轮）。**

## 1. 灰度：电气链已钉死，只剩安装朝向

### 1.1 位序（原 H2 存疑项 → 电气部分已解决）

网表「12路灰度」14P 连接器与固件 `PIN_IN1..IN12` **一一直通，零交叉**：

| 连接器脚 | 网络 | 固件 | | 连接器脚 | 网络 | 固件 |
|---|---|---|---|---|---|---|
| 1 | PB27 | IN1 = bit0 | | 7 | PB17 | IN7 = bit6 |
| 2 | PB12 | IN2 = bit1 | | 8 | PB19 | IN8 = bit7 |
| 3 | PB13 | IN3 = bit2 | | 9 | PB21 | IN9 = bit8 |
| 4 | PB8 | IN4 = bit3 | | 10 | PB14 | IN10 = bit9 |
| 5 | PB20 | IN5 = bit4 | | 11 | PB0 | IN11 = bit10 |
| 6 | PB26 | IN6 = bit5 | | 12 | PB24 | IN12 = bit11 |

（13 = GND，14 = 5V。）

**给硬件组的安装定义**：模组上与排线**第 1 脚**导通的探头（厂商丝印 **P1/G1**）朝**车左**装。
这样固件 `Gray_ReadDarkBitmap()` 的 bit0 = 最左，`board.syscfg` 注释「IN1=最左」成立，
未来循迹权重表不需要任何反转。

注意：厂商手册 p.34 自己的图约定 P1=最右——那只是他们文档的画法，不约束安装朝向；
按上面的定义装即可。若机械结构只允许反着装（P1 在右），**不改 Driver**，反转吸收进未来
Middleware 权重表（唯一修正点），并回报。

装好上电后仍按 `docs/12路灰度传感器配置指南.md` §4 做一分钟实测（黑线放最左，看 bit0）收尾——
排线直通（板端脚 1 ↔ 模组 P1）是本结论唯一未经网表覆盖的一段，万用表一响就能确认。

### 1.2 供电（原 H1 存疑项 → 风险坐实，需硬件组改）

网表实锤：**灰度.14 接在 5V 网络上**（与降压器输入、陀螺仪同网）。

- 手册 p.31：5V 供电时模组输出高约 **4.0V**。
- 12 路信号全部进 MSPM0G3519 的 **PB 常规 IO**。器件常规 IO 的绝对最大输入是 VDD+0.3（≈3.6V），
  只有 PA0/PA1 这类 ODIO 开漏脚是 5V 耐受——**没有一路灰度在 ODIO 上**。4.0V 灌 PB = 超绝对最大值。
  （此为器件手册事实，请硬件组照 G3519 数据手册再核一遍后动手。）
- **要求**：灰度供电从 5V 网改为 **3.3V**（模组支持 3.3–5V，手册 p.10；推荐安装高度 10–30mm
  下探测距离足够），或 12 路每路串 **300–1000Ω**（手册 p.31 原话）。
- **改完之前，12 根信号线不接主控。**

## 3. ⚠ 新发现（比灰度更要紧）：电机与编码器的左右在插座上是交叉的

### 事实链（每一环都有出处）

1. 网表两个 6P 电机插座：
   - **U34** = 电机线 **AO1/AO2**（A 桥）+ 编码器线 **PB7/PB9**
   - **U33** = 电机线 **BO1/BO2**（B 桥）+ 编码器线 **PB10/PB11**
2. 电机驱动模块辨识：H1 排 8 脚序 = `12V(VM), 5V(VCC), GND, AO1, AO2, BO2, BO1, GND`，
   与经典 TB6612 分线板输出侧 **8 脚全中**；对应 H2 排 = `GND, PWMB(PA22), BIN2(PA24),
   BIN1(PA17), STBY(5V), AIN1(PA13), AIN2(PA12), PWMA(PB2)`，方向脚 4/4 与 syscfg 命名自洽。
3. 固件（`motor_hw.c:38-62` + syscfg）：
   - **MOTOR_LEFT** = BIN1/BIN2 + PWM PA22（=PWMB）→ 驱动 **B 桥 → BO1/BO2 → U33**
   - **MOTOR_RIGHT** = AIN1/AIN2 + PWM PB2（=PWMA）→ 驱动 **A 桥 → AO1/AO2 → U34**
   - **QEI_LEFT** = PB7/PB9 → 在 **U34**；**QEI_RIGHT** = PB10/PB11 → 在 **U33**

### 结论

| 插座 | 电机被固件叫作 | 编码器被固件叫作 |
|---|---|---|
| U33 | **左** | **右** |
| U34 | **右** | **左** |

**同一插座上电机与编码器的左右名字相反。** 一只轮子的电机+编码器合插一个 6P 座后，
速度闭环拿到的反馈永远来自**另一只**轮子——左右 PID 交叉耦合，车必然不对，且症状诡异难查。

### 单点修正方案（二选一，等你拍板，未拍板前固件一个字不动）

| 方案 | 定义 | 固件改动 | 备注 |
|---|---|---|---|
| **a（推荐）** | **U33 = 左轮插座**，U34 = 右轮 | 只在 `board.syscfg` 对调 QEI_LEFT/QEI_RIGHT 的 `$name`（编码器命名随电机走）；`encoder.c` 走 `QEI_*_INST` 宏零改动 | 改动面最小，syscfg 本就声明「左右语义由 $name 承载」 |
| b | **U34 = 左轮插座** | 对调电机侧：`motor_hw.c` 的 AIN/BIN 映射 + 左右 PWM 实例对应 | 改动面更大，动的是有 7 项主机测试的模块 |

前提确认（硬件组一句话的事）：驱动模块确为经典脚序的 TB6612 分线板、按 H1/H2 丝印方向插。
确认 + 拍板后固件当轮改完；**H3 手推实测（左轮向前速度为正）在上车时照做**，那是对
`s_direction_sign` 的标定，与本节的插座命名是两件事，互不替代。

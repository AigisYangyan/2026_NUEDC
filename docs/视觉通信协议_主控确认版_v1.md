# 视觉 ↔ 主控 通信协议（主控确认版 v1.0）

> 本文件为**主控端已实现并锁定**的协议，视觉组请以此为准统一实现。

## 0. 一句话

一条 UART 双向链路，跑**两种帧**：运行时视觉往主控发**坐标**（协议 A），开跑前双方用**选题握手**（协议 B）对题号。

## 1. 物理层

| 项 | 值 |
|---|---|
| 接口 | UART，**230400** bps，8 数据位 / 无校验 / 1 停止位（8N1） |
| 接线 | 视觉 TX → 主控 RX，视觉 RX → 主控 TX，**必须共地** |
| 字节序 | 多字节字段一律**小端**（little-endian） |

## 2. 协议 A —— 坐标帧（视觉 → 主控，运行期）

```
┌────────┬────────┬─────────────────────────┬──────────┐
│ 0xAA55 │ 长度1B │ Payload                 │ CRC16 2B │
│ 帧头   │ =9     │ 0x01 + X(f32) + Y(f32)  │ 小端     │
└────────┴────────┴─────────────────────────┴──────────┘
```

| 字段 | 字节 | 说明 |
|---|---|---|
| 帧头 | `AA 55` | 固定，**0xAA 在前** |
| 长度 | 1B | = Payload 字节数，坐标帧固定 **9** |
| 命令 | 1B | `0x01` = 坐标 |
| X | 4B | float32 小端 |
| Y | 4B | float32 小端 |
| CRC | 2B | CRC16-MODBUS，小端，**校验范围 = 长度字节 + Payload**（不含帧头） |

- 整帧 **14 字节**。
- CRC16 算法 = 标准 **MODBUS**（初值 `0xFFFF`，多项式 `0xA001`），即贵组现有 `crc16_modbus` 一致。

## 3. 协议 B —— 选题 / 握手帧（双向，开跑前）

```
┌──────┬──────────┬──────────┬──────┐
│ 0xFF │ 主任务号 │ 子任务号 │ 0xFE │   固定 4 字节，无校验
└──────┴──────────┴──────────┴──────┘
```

- **主控 → 视觉**：下发【主任务号 + 子任务号】，告诉视觉这一题用哪套算法。
- **视觉 → 主控**：**原样回传同一帧**作为确认。
- 主控**收到确认帧后才开始运行**；收不到就不跑（防止视觉没对上题、小车却盲跑）。
- 中间**不带任何数据字段**，只有两个任务号。

## 4. 坐标定义

- `X` / `Y` = 目标中心在图像中的**像素坐标**，原点**图像左上角**，X 向右为正、Y 向下为正。
- ⚠️ **需双方一致的唯一一项**：图像分辨率（用于主控算中心）。当前主控按 **640 × 480**（中心 320,240）。若视觉实际分辨率不同，请在此告知，主控改一个常量即可。

## 5. 运行铁律（务必遵守，否则解析会乱）

1. **运行期只发协议 A 坐标帧**，一种包，不许混发别的包。
2. **运行期不要往坐标流里夹任何"回馈帧 / 状态帧"** —— 高速坐标流里混几帧异类包，主控极易解析失败。
3. **协议 B 握手只在开跑前用一次**，运行中不再发。
4. float 一律 **4 字节小端**。
5. 帧与帧之间连续发送即可，无需额外间隔字节。

## 6. 黄金样例帧（视觉组自校用）

| 含义 | 完整帧（HEX） |
|---|---|
| 坐标 X=100.5, Y=200.3 | `AA 55 09 01 00 00 C9 42 CD 4C 48 43 4C A4` |
| 坐标 X=0.0, Y=0.0 | `AA 55 09 01 00 00 00 00 00 00 00 00 AD B8` |
| 选题 主任务=2, 子任务=1 | `FF 02 01 FE` |
| 视觉回该选题的确认 | `FF 02 01 FE` |

> 自校方法：用你的 CRC16-MODBUS 对第 3～12 字节（`09 01 … 48 43`，即 长度+Payload 共 10 字节）算校验，
> 结果应 = `0xA44C`，上线时低字节在前 → `4C A4`。对得上即实现一致。

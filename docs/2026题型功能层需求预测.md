# 2026 题型 → 功能层需求预测（赛前预制的施工依据）

**定位**：把《2026赛题预测与硬件差距清单》§3.5 的加权题型逐个翻译到**功能层**
（Driver / Middleware / App Service——即 T01 赛题 Task 之下的全部代码）。
用法：赛前把本文所有 P 级缺口写完，赛日无论抽中哪个形态，**只写 Task 段表与编排**。

**边界声明**（守 AGENTS.md §8.3/§15）：预制以「器件能力 + 已确认需求（Q8/Q9 登记、
视觉协议共识）」为界；概率低且无法用器件能力解释的模块只登记「抽中日速建路径」，
不预建接口。每个缺口项开工时仍走 embedded-closed-loop 契约流程——本文是需求预测，
不是契约。

## 1. 逐题型功能层分解

### 形态 A：送药/送货式小车（车题形态 0.45，P(车题)≈1，头名）

预期题面：循迹网格 + 岔路决策 + 视觉识别标识（数字/颜色/图形）+ 定点装卸载荷
（电子秤验收质量）+ 声光提示 +（发挥）双车无线接力。

| 功能 | 归层 | 现状 |
|---|---|---|
| 循迹 + 岔路检测（左岔/右岔/横线/断线） | line_follow + track_elements | **[已有]** |
| 段表编排（FOLLOW_UNTIL 元素→转向→继续） | route | **[已有]** |
| 标识识别结果接收（类别+置信度，非坐标） | driver/uart_vision **协议扩帧** | **[缺口 P2]**：现协议只有坐标帧+握手帧；需与视觉组共识「识别结果帧」（建议：cmd 字段扩一种 payload=类别 u8+置信 u8+附数 u8，沿用 0xAA55+CRC16 帧壳） |
| 识别结果消费（去抖/最新值/时效） | app/service（消费面，归属届时定：route 触发源 or 独立薄 service） | **[缺口 P2 后半]** |
| 定点停车（元素/标识触发 + 定距离修正） | route + motion(剖面直行) | **[已有]** |
| 装卸——**基础线大概率人工放取**（2021F 原样；复杂度校准：TI 杯做减法） | 载荷在位检测（微动/光电，board_gpio 一路）+ 红绿灯指示（beacon 顺带） | **[缺口微 P5'，随 P1 顺带]** |
| 自动装卸（电磁铁/舱门）——**发挥项，非基础线** | driver/electromagnet（P4——主归属已移簇 2 球管，此处仅复用） | **[对冲级，不为 A 预建编排]** |
| 声光提示（到点响一声/装卸红绿灯） | **driver/beacon + service 包装**（Q9） | **[缺口 P1]** |
| 双车接力（呼叫/应答/交接） | **driver/uart_wireless**（透传+帧校验+心跳超时）+ **app/service/link**（协同状态机） | **[缺口 P3]**（Q8；心跳断→停车是 §8.1 硬项） |
| 现场调参（增益/距离/速度） | param_tune TUNE/DRIVE + VOFA | **[已有]** |

### 形态 B：车载云台复合（**R11 出清：≈0，不备赛仅存档**）

> 机电道具必上清单（2025E 连感光纸都列），2026 云台/激光/感光纸全缺席；「小电机
> 自建云台」兜底被 2025 清单反例否定（有小电机条目仍单列云台）。本形态**零预制项、
> 零备赛动作**；gimbal 链封存为已建成资产（启用剧本存档于想定文档）。
> 视觉备赛重心 = 识别管线（P2），不是云台伺服。

预期题面：车行驶中识别目标 → 云台瞄准/指示/投放；角度验收（量角器）。

| 功能 | 归层 | 现状 |
|---|---|---|
| 底盘行驶（同 A/C 全套） | 底盘链 | **[已有]** |
| 云台瞄准闭环 + 调参 | gimbal + vision_aim + tuning(W8) | **[已有]** |
| 行驶中瞄准的运动前馈 | gimbal 读 Odometry_GetPose（契约 §21.3 已留接入点） | **[不预建，速建路径]**：接入点已留，抽中半天接线 |
| 车/云台同拍共存（调度） | scheduler 条目内双服务泵送 | **[已有]**（route 每拍≤1 子服务的先例扩一条即可，届时 Task 层解决） |
| 目标识别帧 | 同 A 的 P2 | **[缺口 P2]** |
| 投放机构 | 同 A 的 P4/P5 | **[缺口 P4/P5]** |

### 形态 C：视觉增强自动行驶（0.25）

2024 H 式：A→B 无标线航位推算 + 半弧线 + 定点停 + 计时圈（+沿途识别）。

| 功能 | 归层 | 现状 |
|---|---|---|
| 无标线航位推算直行/弧线/定角转 | motion + odometry + heading | **[已有]** |
| 计时圈段表（4 圈重复+每点声光） | route 段表（T01 写） | **[已有能力，赛日填表]** |
| 声光 | P1 | **[缺口 P1]** |
| 标定（mm_per_pulse/track_width/pitch） | 既有单一所有者字段 | **[非代码，上车实测]** |

### 形态 D：无线充电循迹车（0.05——2026 清单无超级电容，降权）

底盘链 **[已有]** 全覆盖；无线电能收发归硬件组（非 MCU 功能层）。唯一可能缺口：
电量/效率显示（ADC 读电压电流）——**不预建**，抽中日按需（半天量级，driver/adc 一路）。

### 形态 E：双车跟随全题（0.10）

**清单反证**：2026 附件无超声收发（2025 高职清单有）、无 UWB——跟随题的测距独门
道具缺席，维持低权。若出：测距只能靠视觉（摄像头看前车标识→uart_vision 坐标链复用）
或无线 RSSI（不可靠）。功能层缺口=P3（无线链路）+ 距离估计消费（视觉坐标→距离的
标定式换算，middleware 一个纯函数，抽中日速建）。

### 形态 J：空地协同变体（车题形态 0.15——车规与 2023G 逐字同源）

2023G 复刻变体：无人机侦察（图传+四旋翼，非本队）→ 车接收坐标/指令前往执行。
车侧功能 = 底盘链全套 **[已有]** + 无线链路（**P3 同缺口**）+ 坐标行驶（motion/odometry
**[已有]**）+ 车载显示（oled **[已有]**；≥6 吋屏若归 MCU 则串口屏 driver，**不预建**，
届时半天）+ 声光（P1）。**除 P1/P3 外无新增缺口**——与形态 A/E 完全同押。

### 形态 F：管道钢球位置控制（装置槽 0.50，头名）

预期题面：钢球(10mm)在 PPR 管(内径13.2)内定位/往返/稳住；电磁铁隔管驱动或
倾角驱动；角度/位置验收。

| 功能 | 归层 | 现状 |
|---|---|---|
| 球位置反馈 | 首选摄像头（视觉组看管） → uart_vision 坐标帧 | **[已有]**（坐标链原样复用！） |
| 位置环算法 | middleware/pid（位置式+微分滤波现成） | **[已有]** |
| 执行器①：电磁铁 PWM 吸力 | 同 P4 driver/electromagnet（PWM 档能力） | **[缺口 P4 兜住]** |
| 执行器②：管倾角（小电机/舵机） | driver 新建 | **[不预建]**：执行器形态题面才定 |
| 球位置环 service | 新 service | **[不预建，登记速建路径]**：fake 视觉+真 pid 主机测试半天可立（W8 调参模式直接套） |
| 调参 | tuning 加 profile（W8 模板五钩子） | **[模式已有]** |

### 形态 G：纯视觉识别/测量装置（装置槽 0.30）

R11：识别/被测对象不上清单，摄像头+图传+大屏+量具即完整签名（2025 C 先例）。
测量/识别算力全在视觉上位机；图传独立于 MCU。MCU 功能层只承担：识别结果消费
（**P2 识别帧**——与形态 A 同一缺口，双押）+ 结果显示（oled **[已有]**；≥6 吋
大屏若归 MCU 则串口屏 driver——**不预建**，届时半天）+ 声光（P1）。
**除 P1/P2 外无新增缺口。**

### 形态 H：磁悬浮/电磁类（装置槽 0.08——2026 无霍尔无柱状铁芯）与 形态 I：独立云台瞄准（≈0）

H：模拟电路为主，抽中即重开，**不预制**。I：gimbal 链 **[已有]** 零新增。

## 2. 功能层预制清单（跨题型去重 + 加权优先级）

得分 = Σ(需要它的题型权重)。**这是「先把开发层写完」的施工顺序。**

| 优先 | 模块 | 层 | 得分 | 能力定义（三问式） | 量级 |
|---|---|---|---|---|---|
| **P1** | beacon 声光 | Driver + Service 薄包装 | ≈0.9+（车题全形态+装置槽通用） | 抽象=「响一声/亮红绿/节拍闪」；隐藏=引脚/PWM/定时；归层=driver/beacon + hmi 或独立 service 包装（Q9 归属届时定） | 半天（引脚定案后） |
| **P2** | 视觉识别结果帧 | driver/uart_vision 契约修订 + 消费面 | ≈0.9（A 0.45 + C 沿途识别 0.25 + 装置槽 G 0.30 **多押**，与 P1 并列最高杠杆） | 抽象=「读最新识别结果(类别,置信,seq)」；隐藏=帧壳/CRC/分帧（沿用现有）；归层=uart_vision 扩一种帧型，消费归上层主动查询 | 1 天（**先与视觉组冻结帧格式**——这是唯一有外部依赖的项，最早启动） |
| **P3** | 无线链路 | driver/uart_wireless + service/link 骨架 | ≈0.6（A 发挥 + E 0.10 + J 0.15） | 抽象=「发一帧/收最新帧/链路活着吗」；隐藏=透传模组差异/校验/心跳记账；归层=driver 收发+心跳，service 只做协同状态机；**心跳断→双车停（§8.1）** | 1.5 天（模块到货后；串口资源先定——UART0 现为 BSL） |
| **P4** | 电磁铁执行器 | driver/electromagnet | ≈0.5（**主归属簇 2 球管**（装置槽 0.50 头名）+ A 发挥项复用） | 抽象=「吸持/释放/设吸力档(PWM)」——PWM 吸力档是球管制动的核心能力；隐藏=GPIO/PWM/续流细节；失电行为与机构组确认；最大吸合时长可配 | 半天 |
| P5' | 载荷在位检测+装卸指示 | driver/board_gpio 扩一路 + beacon 复用 | ≈0.43（A 基础线） | 抽象=「载荷在不在」+红绿灯；**基础线人工装卸（2021F 原样），不建 payload service 编排**（复杂度校准，§8.3） | 2 小时（随 P1 顺带） |
| 登记 | gimbal 运动前馈（形态 B ≈0）/ ball_loop 球位置环 service / 距离估计纯函数 / adc 电量 / 串口屏 | 各层 | <0.3 且形态未定 | **不预建**（§8.3）；速建路径已在 §1 各形态注明（均≤1 天，依赖项 P4/pid/uart_vision 届时全就绪；前馈接入点契约 §21.3 已留，抽中瞄准要素半天接线） | — |

**排期建议**（含外部依赖排序）：P2 协议冻结（找视觉组，
最早今天）→ P1 声光+P5' 在位检测（等引脚定案，可并行，合计一天内）→ P3 无线
（等模块到货+串口规划）→ P4 电磁铁（球管对冲，等采购）。
全部完成后，功能层对 §3.5 加权题型的期望覆盖率 ≈ **车题全形态 ≈1（五形态全部踩在
底盘链上）+ 装置槽 0.80（F 球管 0.50 速建路径 + G 纯视觉 0.30 除 P1/P2 零缺口）**——
赛日剩余工作 = T01 段表 + 现场标定 + 机构联调。

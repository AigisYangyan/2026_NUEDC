# 2026 题型 → 功能层需求预测（赛前预制的施工依据）

**定位**：把《2026赛题预测与硬件差距清单》§3.5 的加权题型逐个翻译到**功能层**
（Driver / Middleware / App Service——即 T01 赛题 Task 之下的全部代码）。
用法：赛前把本文所有 P 级缺口写完，赛日无论抽中哪个形态，**只写 Task 段表与编排**。

**边界声明**（守 AGENTS.md §8.3/§15）：预制以「器件能力 + 已确认需求（Q8/Q9 登记、
视觉协议共识）」为界；概率低且无法用器件能力解释的模块只登记「抽中日速建路径」，
不预建接口。每个缺口项开工时仍走 embedded-closed-loop 契约流程——本文是需求预测，
不是契约。

## 0. 现有功能层资产盘点（2026-07-21，全部构建绿）

| 层 | 已有模块 | 能力一句话 |
|---|---|---|
| Driver | motor(TB6612+slew/换向/超时) / encoder(QEI) / gray(12路) / imu(Yaw/GyroZ) / key / oled / uart_vofa / uart_vision(坐标+握手) / stepmotor_uart / emm42 / clock / param_store(flash) / bsl_entry | 全部执行器与传感链路 |
| Middleware | pid / track_error / track_elements(断线/横线/左右岔) / speed_plan / move_profile / odometry+heading / vision_aim(PD) | 全部纯算法 |

## 1. 逐题型功能层分解

### 形态 A：送药/送货式小车（车题 0.40 × P(车)0.9 ≈ 0.36，头名）

预期题面：循迹网格 + 岔路决策 + 视觉识别标识（数字/颜色/图形）+ 定点装卸载荷
（电子秤验收质量）+ 声光提示 +（发挥）双车无线接力。

| 功能 | 归层 | 现状 |
|---|---|---|
| 循迹 + 岔路检测（左岔/右岔/横线/断线） | line_follow + track_elements | **[已有]** |
| 段表编排（FOLLOW_UNTIL 元素→转向→继续） | route | **[已有]** |
| 标识识别结果接收（类别+置信度，非坐标） | driver/uart_vision **协议扩帧** | **[缺口 P2]**：现协议只有坐标帧+握手帧；需与视觉组共识「识别结果帧」（建议：cmd 字段扩一种 payload=类别 u8+置信 u8+附数 u8，沿用 0xAA55+CRC16 帧壳） |
| 识别结果消费（去抖/最新值/时效） | app/service（消费面，归属届时定：route 触发源 or 独立薄 service） | **[缺口 P2 后半]** |
| 定点停车（元素/标识触发 + 定距离修正） | route + motion(剖面直行) | **[已有]** |
| 装卸执行（电磁铁吸/放 或舵机爪） | **driver/electromagnet**（GPIO/PWM 吸持-释放-超时安全释放） | **[缺口 P4]** |
| 装卸编排（到位→装→确认→走） | **app/service/payload**（装/卸/状态查询三能力） | **[缺口 P5]** |
| 载荷在位检测（微动/光电） | driver/board_gpio 扩一路输入 | **[缺口小，随 P5]** |
| 声光提示（到点响一声/装卸红绿灯） | **driver/beacon + service 包装**（Q9） | **[缺口 P1]** |
| 双车接力（呼叫/应答/交接） | **driver/uart_wireless**（透传+帧校验+心跳超时）+ **app/service/link**（协同状态机） | **[缺口 P3]**（Q8；心跳断→停车是 §8.1 硬项） |
| 现场调参（增益/距离/速度） | param_tune TUNE/DRIVE + VOFA | **[已有]** |

### 形态 B：车载云台复合（0.22 × 0.9 ≈ 0.20）

预期题面：车行驶中识别目标 → 云台瞄准/指示/投放；角度验收（量角器）。

| 功能 | 归层 | 现状 |
|---|---|---|
| 底盘行驶（同 A/C 全套） | 底盘链 | **[已有]** |
| 云台瞄准闭环 + 调参 | gimbal + vision_aim + tuning(W8) | **[已有]** |
| 行驶中瞄准的运动前馈 | gimbal 读 Odometry_GetPose（契约 §21.3 已留接入点） | **[缺口 P6]**：契约修订接线，量小 |
| 车/云台同拍共存（调度） | scheduler 条目内双服务泵送 | **[已有]**（route 每拍≤1 子服务的先例扩一条即可，届时 Task 层解决） |
| 目标识别帧 | 同 A 的 P2 | **[缺口 P2]** |
| 投放机构 | 同 A 的 P4/P5 | **[缺口 P4/P5]** |

### 形态 C：视觉增强自动行驶（0.18 × 0.9 ≈ 0.16）

2024 H 式：A→B 无标线航位推算 + 半弧线 + 定点停 + 计时圈（+沿途识别）。

| 功能 | 归层 | 现状 |
|---|---|---|
| 无标线航位推算直行/弧线/定角转 | motion + odometry + heading | **[已有]** |
| 计时圈段表（4 圈重复+每点声光） | route 段表（T01 写） | **[已有能力，赛日填表]** |
| 声光 | P1 | **[缺口 P1]** |
| 标定（mm_per_pulse/track_width/pitch） | 既有单一所有者字段 | **[非代码，上车实测]** |

### 形态 D：无线充电循迹车（0.10 × 0.9 ≈ 0.09）

底盘链 **[已有]** 全覆盖；无线电能收发归硬件组（非 MCU 功能层）。唯一可能缺口：
电量/效率显示（ADC 读电压电流）——**不预建**，抽中日按需（半天量级，driver/adc 一路）。

### 形态 E：双车跟随全题（0.10 × 0.9 ≈ 0.09）

**清单反证**：2026 附件无超声收发（2025 高职清单有）、无 UWB——跟随题的测距独门
道具缺席，维持低权。若出：测距只能靠视觉（摄像头看前车标识→uart_vision 坐标链复用）
或无线 RSSI（不可靠）。功能层缺口=P3（无线链路）+ 距离估计消费（视觉坐标→距离的
标定式换算，middleware 一个纯函数，抽中日速建）。

### 形态 F：管道钢球位置控制（装置槽 0.30，Tier2 之首）

预期题面：钢球(10mm)在 PPR 管(内径13.2)内定位/往返/稳住；电磁铁隔管驱动或
倾角驱动；角度/位置验收。

| 功能 | 归层 | 现状 |
|---|---|---|
| 球位置反馈 | 首选摄像头（视觉组看管） → uart_vision 坐标帧 | **[已有]**（坐标链原样复用！） |
| 位置环算法 | middleware/pid（位置式+微分滤波现成） | **[已有]** |
| 执行器①：电磁铁 PWM 吸力 | 同 P4 driver/electromagnet（PWM 档能力） | **[缺口 P4 兜住]** |
| 执行器②：管倾角（小电机/舵机） | driver 新建 | **[不预建]**：执行器形态题面才定 |
| 球位置环 service | 新 service | **[不预建，登记速建路径]**：fake 视觉+真 pid 主机测试半天可立（W8 调参模式直接套） |
| 调参 | tuning 加 profile（W8 模板五钩子） | **[模式已有]** |

### 形态 G：视觉测量/远程监控装置（装置槽 0.28）

测量/识别算力全在视觉上位机；图传独立于 MCU。MCU 功能层只承担：云台扫描
（gimbal **[已有]**）+ 结果显示（oled **[已有]**；≥6 吋大屏若归 MCU 则串口屏
driver——**不预建**，届时半天）。**功能层缺口≈0，本形态对预制清单无新增。**

### 形态 H：磁悬浮/电磁类（装置槽 0.17）与 形态 I：独立云台瞄准（0.10）

H：模拟电路为主，抽中即重开，**不预制**。I：gimbal 链 **[已有]** 零新增。

## 2. 功能层预制清单（跨题型去重 + 加权优先级）

得分 = Σ(需要它的题型权重)。**这是「先把开发层写完」的施工顺序。**

| 优先 | 模块 | 层 | 得分 | 能力定义（三问式） | 量级 |
|---|---|---|---|---|---|
| **P1** | beacon 声光 | Driver + Service 薄包装 | ≈0.81（A+B+C+装置题通用） | 抽象=「响一声/亮红绿/节拍闪」；隐藏=引脚/PWM/定时；归层=driver/beacon + hmi 或独立 service 包装（Q9 归属届时定） | 半天（引脚定案后） |
| **P2** | 视觉识别结果帧 | driver/uart_vision 契约修订 + 消费面 | ≈0.56（A+B+E） | 抽象=「读最新识别结果(类别,置信,seq)」；隐藏=帧壳/CRC/分帧（沿用现有）；归层=uart_vision 扩一种帧型，消费归上层主动查询 | 1 天（**先与视觉组冻结帧格式**——这是唯一有外部依赖的项，最早启动） |
| **P3** | 无线链路 | driver/uart_wireless + service/link 骨架 | ≈0.45（A 发挥+E） | 抽象=「发一帧/收最新帧/链路活着吗」；隐藏=透传模组差异/校验/心跳记账；归层=driver 收发+心跳，service 只做协同状态机；**心跳断→双车停（§8.1）** | 1.5 天（模块到货后；串口资源先定——UART0 现为 BSL） |
| **P4** | 电磁铁执行器 | driver/electromagnet | ≈0.63（A 装卸 0.36 + F 执行器 0.27，**一个 driver 兜两题型**） | 抽象=「吸持/释放/设吸力档(PWM)」；隐藏=GPIO/PWM/续流细节；安全项=上电默认释放?——**注意：装卸场景默认失电释放，需与机构组确认失电行为**；命令超时不适用（保持吸合是合法态，改为最大吸合时长可配） | 半天 |
| **P5** | payload 装卸 | app/service/payload | ≈0.36（A） | 抽象=「装载/卸载/在位查询」；隐藏=执行器时序/在位去抖；归层=service 编排 P4+board_gpio 输入 | 半天 |
| **P6** | gimbal 运动前馈 | gimbal 契约修订（读 Odometry_GetPose） | ≈0.20（B） | 契约 §21.3 已留接入点；折入瞄准的几何随题面定，先接读点+文档 | 半天 |
| 登记 | ball_loop 球位置环 service / 距离估计纯函数 / adc 电量 / 串口屏 | 各层 | <0.3 且形态未定 | **不预建**（§8.3）；速建路径已在 §1 各形态注明（均≤1 天，依赖项 P4/pid/uart_vision 届时全就绪） | — |

**排期建议**（含外部依赖排序）：P2 协议冻结（找视觉组，最早今天）→ P1 声光（等引脚定案，
可并行）→ P4 电磁铁 → P5 payload → P3 无线（等模块到货+串口规划）→ P6 前馈。
全部完成后，功能层对 §3.5 加权题型的期望覆盖率 ≈ **0.9（车题全形态）+ 装置槽 0.58
（F 速建路径 + G 零缺口 + I 已有）**——赛日剩余工作 = T01 段表 + 现场标定 + 机构联调。

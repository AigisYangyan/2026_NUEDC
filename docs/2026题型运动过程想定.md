# 2026 题型运动过程想定（把每个预测题在脑子里放一遍电影）

**定位**：《2026题型功能层需求预测》说"要写什么代码"；本文说"比赛当天场上会发生什么"——
每个形态给出：场地想定图、从上电到收尾的**逐秒运动时间线**、验收动作、易丢分点。
时间线每步末尾的〔〕标注干活的功能层模块。分值结构按历届惯例：基础 60~70 + 发挥 30~40 +

## 形态 A：送药/送货式小车（加权 0.36，头名）

### 场地想定（参照 2021 F 病房布局魔改）

```text
                 ┌─────┐   ┌─────┐
                 │房间5 │   │房间6 │        远端：需两次岔路决策
                 └──┬──┘   └──┬──┘
        ════╦═══════╩══════════╩══════  ← 第二岔口(十字/丁字)
            ║
   ┌─────┐ ║ ┌─────┐
   │房间3 │═╬═│房间4 │                   中部：一次岔路决策
   └─────┘ ║ └─────┘
        ════╬════  ← 第一岔口
            ║
   ┌─────┐ ║ ┌─────┐
   │房间1 │═╬═│房间2 │                   近端：直走可达
   └─────┘ ║ └─────┘
            ║   ← 主走廊（黑线，线宽~1.8cm）
      ┌─────╨─────┐
      │  药房/起点  │  ← 出发区+装载区；标识牌立在各房间门口（数字/图形，位置随机）
      └───────────┘
```

### 运动时间线（一次满分基础趟，想象 90 秒）

1. **T+0s 备赛位**：车摆进药房出发区压线；OLED 菜单选赛题条目，等待发车〔menu/scheduler〕。
2. **T+2s 装载**：裁判把载荷（≈200g 药包/物块）放进载荷仓——或发挥部分由电磁铁自动
   吸取；在位开关触发，**红灯亮 = 已装载**〔payload + electromagnet + beacon〕。
3. **T+5s 识别**：裁判在药房口亮出目标房间标识牌（数字"5"）。摄像头注视 1~2s，
   识别成功**蜂鸣一声**，车记下"去 5 号房"〔uart_vision 识别帧 + beacon〕。
4. **T+8s 出发**：车沿主走廊循迹加速到巡航速度（0.3~0.5 m/s），车身贴线无蛇形
   〔line_follow + speed_plan + chassis〕。
5. **T+15s 第一岔口**：灰度检测到左右岔特征——目标是 5 号（远端），**直行穿过**，
   不减速或轻减速〔track_elements + route FOLLOW_UNTIL〕。
6. **T+25s 第二岔口**：检测到十字——按识别结果**左转**：减速→定角转向 90°→重新
   捕线→加速〔route 段表：FOLLOW_UNTIL(岔)→TURN(90°)→FOLLOW〕。
7. **T+35s 到达 5 号房**：门口横线触发→梯形减速→**停车，车头压横线 ±2cm**→
   **蜂鸣 + 红灯闪** = 到达提示〔track_elements 横线 + motion 剖面 + beacon〕。
8. **T+38s 卸载**：电磁铁释放/舵机开仓，载荷落入房间格；在位开关变空→**绿灯亮**
   = 卸载完成〔payload〕。
9. **T+42s 返程**：原地定角转 180°→沿原路循迹返回，岔口全部直行→药房横线停车→
   **长蜂鸣** = 任务完成，总用时定格〔motion TURN + route 返程段表〕。

### 发挥部分想定（+30~40 分）

- **双车接力**：远端房间(7/8 号)由 1 号车送到中转岔口停下→**无线呼叫** 2 号车出发→
  2 号车到中转点接收载荷（对接/换手）→继续送达→2 号车返回后 1 号车才收尾
  〔uart_wireless + link 心跳；断链双车立停〕。
- **多目标连送**：一次识别两张牌，规划顺路先后送。
- **计时加严**：总时间越短越好。

### 易丢分点

岔口误检（元素置信拍数不足→误转）；识别错房间（直接 0 分趟）；停车过冲压不住横线
（剖面减速参数）；返程 180° 转完捕不回线（recovery 误差幅值）；双车相撞（心跳/互锁没做）。

## 形态 B：车载云台复合（加权 0.20）

### 场地想定

```text
   ┌───────────────────────────────┐
   │   靶标A(高)          靶标B(低)   │   ← 场地四周不同高度/方位立靶标
   │      ╔═══════════════╗        │      （数字牌/色块/同心圆靶纸）
   │      ║   循迹环线     ║ 靶标C   │
   │      ║  ┌─────┐      ║        │
   │ 靶标D ║  │禁行区│      ║        │
   │      ╚══╧═════╧══════╝        │
   │        [出发区]                │
   └───────────────────────────────┘
```

### 运动时间线（想象 120 秒一圈）

1. **T+0s**：车带云台（总高 ≤35cm 内自定）出发区就位；云台回中、步进使能锁轴
   〔gimbal ARMING：enable→preset→清零〕。
2. **T+5s 起跑**：车沿环线循迹行驶；云台进入扫描态，左右摆动搜索视野内靶标
   〔line_follow ∥ gimbal 双服务同拍泵送〕。
3. **T+15s 行进间锁定**：摄像头捕到靶标 A → 视觉持续输出像素坐标 → 云台边走边把
   靶心拉进死区（车在动，瞄准点不断流动——odometry 前馈补车速引起的像素漂移）
   〔vision_aim PD + gimbal 前馈 P6〕。
4. **T+20s 指示区停车**：到达靶标 A 对应的停车横线→停稳 1s 内云台完成精瞄→
   **激光/指示物落靶 ≥3s，蜂鸣一声**——裁判用量角器/靶纸判定指向偏差
   〔motion 停车 + gimbal AIMING 收敛 + beacon〕。
5. **T+30s 续跑**：云台保持锁定或回中，车继续向靶标 B……重复 3-4，一圈内完成
   全部靶标，总时间计分。
6. **发挥**：移动靶（靶标在导轨上动）；行进间不停车完成指示；报靶（识别靶标上的
   数字并显示/播报）〔uart_vision 识别帧〕。

### 易丢分点

车振动让像素噪声变大（云台减振 + DB 死区重标——**GimbalTune 上车重调一轮**）；
行进间瞄准的稳态滞后（前馈没接=追不上）；云台供电压降导致步进丢步；靶标高低差
超出俯仰轴程（travel_limit 与机械安装角要现场核）。

## 形态 C：视觉增强自动行驶（加权 0.16）

### 场地想定（2024 H 原样+识别加料）

```text
        B ●━━━━━━━━━━━● C
         ╱  (半弧 r≈40cm)  ╲        实线段：有黑线引导
        │                  │        虚线段：无线区，纯航位推算
        A ●┈┈┈┈┈┈┈┈┈┈┈● D
            ↑ A-B/C-D 直线段 ~100cm，A 为起点
   （2026 加料想定：沿途立 2~3 块数字牌，经过时须识别并蜂鸣对应次数）
```

### 运动时间线（一圈 ≤30s 级）

1. **T+0s**：A 点压点位放车，方向粗对准 B；发车。
2. **T+0~4s A→B 无线直行**：无引导线——里程计定距 + IMU 航向保持，直行 1m 终点
   偏差 ±5cm 内；**B 点声光一响**〔motion 剖面直行 + heading hold + beacon〕。
3. **T+4~9s B→C 半弧**：定半径圆弧原语，双轮速比走弧，出弧航向对准 C
   〔motion ARC + odometry〕。
4. **T+9~13s C→D 直行 + 沿途识别**：经过数字牌"3"→蜂鸣 3 声不停车〔uart_vision 识别帧〕。
5. **T+13~18s D→A 半弧回起点**：压 A 点停车，**长鸣**，单圈计时定格。
6. **4 圈连跑**：重复 1-5，每圈误差不累计（每点位重新校正）；总时间越短越好。

### 易丢分点

mm_per_pulse/轮距没标定准（弧线出口偏航，一圈错 10cm 四圈错 40cm）；电池掉压后
半程速度环变软（连跑前满电+电压补偿）；无线区地面打滑（出发加速度限制）。

## 形态 F：管道钢球位置控制（装置槽 0.30）

### 场地想定

```text
      摄像头(俯视/侧视整管)
           ▼
   0cm   20   40   60   80cm  ← 管身贴刻度
   ╔══════════════════════╗
   ║  ●球(10mm)            ║   ← PPR 管（内径13.2mm）水平架设
   ╚══════╤═══════════════╝
        [电磁铁]  ← 管下滑轨上可移动，或固定多只
     （变体：管一端由舵机抬升，倾角驱动球滚动）
```

### 运动时间线（想象一次基础题 60 秒）

1. **T+0s**：球放入管口 0cm 处静止；上位机/按键设定目标刻度"60cm"。
2. **T+3s 驱动**：倾角微抬（或电磁铁接力吸引）→球开始滚动，摄像头逐帧回报球心
   坐标〔uart_vision 坐标链原样复用 + pid 位置环〕。
3. **T+8s 制动逼近**：球接近 60cm→反向倾角/电磁铁 PWM 吸力制动→球减速——这里会
   看到一两次小幅来回（欠阻尼），PD 调好则一次进位〔pid + tuning 新 profile 调参〕。
4. **T+12s 到位保持**：球停在 60±1cm，**保持 ≥5s 不动**，蜂鸣宣告〔beacon〕。
5. **发挥**：多点往返（0→60→20→80cm 序列计时）；管倾角突加扰动后重新稳住；
   最快穿越且末端不冲出管口。

### 易丢分点

球速快于视觉帧率（拖影丢跟踪→限最大驱动力）；电磁铁隔管吸力非线性（PWM-吸力先
台架标定）；到位判定抖动（死区+保持计时和云台 W8 同款套路）。

## 形态 E/D/G 速写（合计权重低，一段话想象）

- **E 双车跟随（0.09）**：前车沿 8 字线变速走走停停，后车 20cm 跟距不碰不丢——
  后车视觉盯前车尾标估距，无线心跳断则双停〔link + 距离估计纯函数〕。
- **D 无线充电循迹（0.09）**：车循迹进充电区压线圈停 30s 充电→靠电容电量走完剩余
  半圈→计总时间与电量效率，运动本身就是基础循迹〔底盘链零新增〕。
- **G 视觉测量/监控（0.28 装置槽但运动少）**：云台定点扫描场景→找到目标物→上位机
  测距离/尺寸→大屏显示读数；全程只有云台在动〔gimbal 扫描 + 上位机〕。

## 比赛日通用操作流程（把上面任何一部电影跑起来的固定开场）

1. **上电自检**：OLED 菜单亮 → DEBUG 组过一遍：GrayTest 看 12 路灰度贴地阈值
   （光线变了拧电位器）、EncoderTest 看双轮脉冲、MotorDir 确认方向；
2. **现场标定**：TUNE/DRIVE 组按钮调循迹增益/剖面参数 SAVE 掉电保存；云台题加跑
   GimbalTune（VOFA 命令模式重标 DB/kp/kd——车载后噪声必变）；
3. **段表核对**：赛题 Task 条目 = route 段表 + 识别触发映射，试跑一遍全流程；
4. **发车**：选赛题条目 → 等裁判口令 → GO。

> 立即回补《功能层需求预测》。

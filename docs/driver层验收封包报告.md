## 3. ★ 拿到硬件之后，最主要做的事（按此顺序，不要跳）

Driver 软件侧已经干净，**当前全部瓶颈都在这一节**。四项都是「Driver 无法自证、只有实物能回答」
的问题（`AGENTS.md` §15.4），用厂商上位机 / 万用表 / 手推轮子直接做，不经 App 链路。

> **2026-07-17 封包后更新**：硬件组网表已对照完毕（`docs/网表对照结论.md`）——H1 定案为
> 保持 5V 供电 + 12 路串阻（用户裁定，方案见 `docs/给硬件组的修改方案.md`）；H2 电气链钉死、
> 安装定义已给（G1/P1=车左，驾驶员视角）；M1（U33/U34 电机-编码器左右交叉）**已拍板并由固件
> 吸收**（方案 a：QEI `$name` 对调，现 **QEI_LEFT=TIMG8/PB10/PB11=U33=左轮、
> QEI_RIGHT=TIMG9/PB7/PB9=U34=右轮**——本节及 §2.1 中出现的旧归属均为封包时点的记录）；
> N1/N2/N3（K4 断线、PA0/PA1 上拉、IMU TX 电平）移交硬件组。**以上述两文为准，本节保留原文。**

### 第 0 步（先于一切上电）：H1 灰度供电电压 —— 唯一会烧硬件的项

1. 确认 NCHD1 模组实际供电是 3.3V 还是 5V。
2. 若 5V：手册 p.31 写明输出高电平约 **4.0V**。查 MSPM0G3519 数据手册确认 PB 组 12 个输入脚
   是否 5V 耐压；**不耐压就必须改 3.3V 供电，或每路串 300–1000Ω 限流电阻**。
3. 没做完这一步，**灰度模组的 12 根信号线不许接主控**。烧掉的 IO 换不回来。

### 第 1 步：H4 IMU 上位机配置（不配对，固件读到的就是乱码）

用厂商上位机（详细步骤见 `docs/IMU陀螺仪配置指南.md`）：

1. 波特率 115200 → **230400**（固件 syscfg 已配 230400，两边不一致 = 乱码）。
2. 输出速率 10 Hz → **500 Hz**（10 Hz 下 100 Hz 控制环 90% 的 tick 读陈旧值，内环失效）。
3. **自动零偏校准**：车放平、静止约 20 秒。Yaw 漂的解药是这个，不是提速率。
4. **装平**：Z 轴垂直地面，歪装会把 pitch/roll 混进 yaw。
5. 正方向实测：车逆时针转，记录 `yaw_deg` 增还是减，报告回来——符号修正只做一处。

### 第 2 步：H3 编码器方向标定

1. 手推**左轮向前**，读 `Encoder_GetSnapshot()`，左轮速度应为**正**。
2. `encoder.c:41` 的 `s_direction_sign[] = {-1, 1}` 是**旧板**标定值；接线正确时预期改 `{1, 1}`
   ——**但未实测前一个字符都不许改**（§8.1 禁止猜方向）。
   ⚠ 时点注记：本步引脚归属以 QEI 改名后为准——**左 = PB10(A)/PB11(B) = U33，
   右 = PB7(A)/PB9(B) = U34**（`给硬件组的修改方案.md` §4.1），不是封包时点写的 PB7/PB9=左。
3. ⚠ 修正点全仓只有 `s_direction_sign` 这一个。**任何地方加第二个反转开关都会与它抵消**，
   变成「两处都改车是对的、改任何一处车反着走」的极难排查故障。

### 第 3 步：H2 灰度位序实测

1. 黑线移到阵列**最左**，读 `Gray_ReadDarkBitmap()`，看 bit0 还是 bit11 置位。
2. 厂商约定 P1=最右、本仓 syscfg 注释写 IN1=最左，**二者矛盾且固件无法自证**——实测前任何左右权重表都是猜的。
3. 位序修正只能落在未来 Middleware 的权重表里，**Driver 里不加第二个反转开关**（同 H3 教训）。
4. 顺手做：12 路电位器逐路调（手册 p.23 三步法），安装高度 10–30mm（灯罩底距地）。

### 第 4 步：电机首次带电验证（§8.1 分阶段流程，不可跳级）

1. **架空机构 / 断开负载**，低占空比、短持续时间，确认方向、10 kHz PWM 频率与停止行为。
2. 观测手段用串口 / OLED / 示波器 / 直接可见的硬件行为留下记录；**不得**以「接着调试器单步看过了」作为验收。
3. 确认 `Motor_Update` 的三个安全占位参数是否符合实机：换向死区 5ms、命令超时 100ms、
   slew 100/ms（主机测试标注 pending hardware measurement）——不符则按实测改，改后重跑主机测试。
4. 全部确认后再逐步加输出、上负载。

### 其余移交硬件组（不阻塞上述四步）

- 核心板晶振书面确认（PA3/PA5/PA6 是否实焊）。
- PA0/PA1（VOFA）引出；引出前 VOFA 实物链路不可用，固件侧已就绪。

### 硬件实测结果的回填义务

每完成一项，把结果（方向符号、位序、供电结论、IMU 配置截图/记录）报告回来：
H3 改 `s_direction_sign` 一处；H2 记入未来权重表设计输入；H1/H4 记入
`agent/phase2_driver_rewrite/plan_driver_first_order.md` §4 对应条目并销项。
**改完任何一处后重跑主机测试 + 构建**，证据行入提交。

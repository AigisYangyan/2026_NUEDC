## 0. 先解疑惑：没有 Kp/Ki/Kd，位置环的旋钮就是这 5 个

这条位置环**不是 PID 形式**，是**梯形剖面形式**——所以你在 DRIVE 组里找不到 Kp/Ki/Kd，
但**不是不能调**：位置环的调参旋钮换成了下面这 5 个，全在 DRIVE 组按钮可调 + SAVE 存盘。
它们和 PID 各增益一一对应地起作用：

| DRIVE 旋钮 | 管什么 | 相当于 PID 在调 | 单位（显示值） |
|---|---|---|---|
| `Decel` | 刹停快慢 | **压过冲 / 阻尼**（最关键） | ×1000=m/s²（300=0.30） |
| `Accel` | 加速快慢 | 响应速度 | ×1000=m/s²（300=0.30） |
| `Cruise` | 整体跑多快 | 输出上限 | ×1000=m/s（200=0.20） |
| `Start` | 起步不卡住 | 消死区/静摩擦 | ×1000=m/s（80=0.08） |
| `Dist` | 走多远 | 目标设定值 | mm（1000=100cm） |

**30 秒上手口诀**：
> **冲过头 → 加大 `Decel`；太慢/太肉 → 加大 `Accel` 和 `Cruise`；起步卡住 → 加大 `Start`；停不到位差最后一点 → 略减 `Decel` 或加大 `Start`。改完点 `SAVE`。**

**为什么不用 PID**（你 7/19 拍的裁定）：按距离参数化的梯形剖面**本身就是位置闭环**（每拍拿实测距离重算速度、减速段自然收敛到 0），再叠一个位置 PID 就是第二个"从距离算速度"的所有者，会和剖面打架（§8.2），还会引入积分颠振/过冲。**剖面停得比裸 PID 更稳、更准**——这 5 个旋钮就是它完整的现场调参面。

> 想改回经典 Kp/Ki/Kd 位置 PID 也能做（推翻上面裁定），但当前不需要——先用这 5 个调，手感一点不少。

## 1. 这是什么

「走一段指定距离后停准」的位置环。结构是**前馈 + 位置闭环合一**：

```
速度  ┌── 匀速 cruise ──┐
     /                  \        v(s) = min( cruise,
  加速                    减速          √(start² + 2·accel·s),   ← 加速段
   /                        \           √(2·decel·rem) )         ← 减速段→0
──┴──────────────────────────┴──→ 已行进距离 s
  start                      0        s=已行进, rem=剩余=target−s
```

- **按「距离」参数化**，不是按时间。每拍拿里程计实测的已行进距离 `s` 重算该给多快 →
  **前馈本身就是位置反馈**，减速段随剩余距离自然收敛到 0，到位停准。
- **没有单独的纵向 PID**（裁定）：剖面已经把位置闭环做了，再叠一个 PID 就是第二个「从距离算速度」的所有者，会互相打架。
- **横向（走直）另说**：若开 `heading_hold`，用一个独立的航向保持 PID 纠偏保持直线——那是横向、与纵向剖面正交，两码事。

对照旧的 `Motion_StartStraight`（恒速版）：恒速版全程一个速度、到点急停；本原语平滑加速-匀速-减速，停得稳、停得准。**旧恒速版保留不动**，段表按需二选一。

## 2. 板载入口与菜单操作（已接线，§28）

**已上板，和 LineFollow 同款闭环**：DEBUG 组有运行条目、DRIVE 组按钮调参 + SAVE 掉电存。
> 对应代码：`hc-team/app/system/app_compose.c`（`ProfiledStraight` 条目 + `DRIVE` 组）+ param_tune schema2。

### 按键映射（固定）

| 按键 | 作用 |
|---|---|
| K1 | 上移 / 数值 **加** |
| K2 | 下移 / 数值 **减** |
| K3 | 进入 / **确认**（在 SAVE 项上 = 存盘） |
| K4 | 返回 / **退出运行条目**（退出即安全停车） |

### 跑一次定长直行

进 `DEBUG` 组 → 选 `ProfiledStraight` → K3 进入。小车按当前剖面参数跑 `Dist`（默认 1000mm=100cm）后**停在目标**。
K4 退出即刹停。**要再跑一次：K4 退出、再 K3 进入**（每次进入重跑一段）。

> ⚠ **前置：内环速度环得先有增益，否则车不动。** 底盘速度环（轮速 PID）默认增益是 **0**，装配层开机不设——
> 意思是**没先调过速度环，进 ProfiledStraight 命令了速度也出不了力、车不动**（好处是新板第一次不会堵转）。
> 先用 `DEBUG/SpeedTune` 把左右轮速度环调出增益，再回来跑 ProfiledStraight。（速度环增益目前不持久化，掉电要重调——这是已知局限。）

### 调参 + 存盘（DRIVE 组）

进 `DRIVE` 组（进去界面标题显示 `-- PARAMS --`）→ 选参数 → K3 进编辑 → K1/K2 加减 → K4 回列表 → 选 `SAVE` → K3 存 flash。

| DRIVE 项 | 单位 | 说明 |
|---|---|---|
| `Dist mm` | mm | 测试运行距离（默认 1000） |
| `Cruise` | ×1000 = m/s（显示 200 = 0.20） | 匀速上限 |
| `Start` | ×1000 = m/s | 起步速 |
| `Accel` | ×1000 = m/s² | 加速度 |
| `Decel` | ×1000 = m/s² | 减速度 |
| `SAVE` | — | K3 一次性写片内 flash（掉电保存） |

- **编辑只改内存，存盘要点 SAVE**。不点 SAVE 就掉电会丢、回默认。
- **循迹增益在另一组 `TUNE`**，与 DRIVE 各调各的；但**共享同一 flash blob**——任一组按 SAVE 都会把两组当前值一起存盘。
- 现场闭环：**选 ProfiledStraight 跑 → K4 退 → DRIVE 改参 SAVE → 再选它跑 → 看效果**，不用改代码烧录。

> **首次上板注意（schema 升级）**：本版把 flash 记录从旧 13B 升到 33B，**旧版存的循迹增益会一次性失效**（读不出→回默认）。上板后循迹和本条目都要**重新调一遍 + SAVE**。
> **默认剖面保守能跑**：出厂默认 cruise 0.20 / start 0.08 / accel 0.30 / decel 0.30，第一次进就能慢慢跑约 1m 停下（不像 LineFollow 默认 0 增益）。但 `mm_per_pulse` 仍是占位——**距离标定前别当真**（见步骤 0），首次务必架空/低速验方向与停止。

## 3. 参数清单（都在 `Motion_Config_T`）

| 字段 | 单位 | 含义 | 约束 |
|---|---|---|---|
| `profile_cruise_mps` | m/s | 匀速段速度上限（你想让它多快跑） | > 0 |
| `profile_start_mps` | m/s | 加速段起步速——脱离静摩擦、车真能动起来的最低速 | 0 ≤ start ≤ cruise |
| `profile_accel_mps2` | m/s² | 加速度（起步爬到 cruise 有多快） | > 0 |
| `profile_decel_mps2` | m/s² | 减速度（**决定刹停距离和会不会冲过头**） | > 0 |
| `straight_tol_mm` | mm | 到位容差（差多少算「到了」） | > 0 |

**前置依赖（不在剖面里，但位置环准不准的根基）：**

| 字段 | 含义 |
|---|---|
| `mm_per_pulse` | 编码器一个脉冲 = 轮面前进多少 mm。**距离测不准，位置环全盘皆错**——先标它。唯一所有者是 odometry cfg。 |
| `hold_kp/ki/kd`、`hold_diff_limit_mps` | 航向保持 PID（仅 `heading_hold=true` 时用，管走直，不管走多远） |

## 4. 调参顺序（按这个来，别跳步）

### 步骤 0：先标 `mm_per_pulse`（位置环的地基）

让小车走一段**已知的物理距离**（拿卷尺量，比如地上标 1000 mm），读里程计报告的行进量，反算：
`mm_per_pulse_新 = mm_per_pulse_旧 × 实际物理距离 / 里程报告距离`。
标到「让它走 1000 mm，里程也报≈1000 mm」为止。**这一步不准，后面全白调。**

### 步骤 1：找 `start_mps`（起步速）

架空或低速，从很小的速度往上试，找到**轮子真能稳定转起来、车能动**的最低速度。
填这个值当 `start_mps`。太低 → 起步阶段车卡着不动；太高 → 起步一顿冲。

### 步骤 2：定 `cruise_mps`（巡航速）

你希望的正常行驶速度上限。**第一次上板保守**，先低速（比如 0.3 m/s）跑通全流程，稳了再提。

### 步骤 3：定 `accel` / `decel`

- `accel`：起步爬升快慢，从机构能力起，太大起步打滑、太小加速肉。
- `decel`：**最关键** —— 直接决定刹停距离和会不会冲过头。见第 5 节公式算初值，然后实测微调：
  冲过头 → 加大 `decel`；停得太急打滑/点头 → 减小 `decel`。

### 步骤 4：定 `straight_tol_mm`（到位容差）

设成你能接受的停准误差（比如 ±3 mm）。太小 → 终段慢速下可能迟迟差最后 1 mm 不触发到位（靠超时兜底）；太大 → 停得早、不够准。

### 步骤 5：走直 —— `heading_hold` 的航向 PID（可选）

若开 `heading_hold=true`：先只加 `hold_kp` 到能压住跑偏，再少量 `hold_kd` 压抖，`hold_ki` 一般不加。
`hold_diff_limit_mps` 是纠偏差速上限，越大转向越猛。**注意第 7 节的终段提示**。

### 步骤 6：验停准 + 逐步放开

短程低速先验「加速-匀速-减速-停准」四段都对、停在容差内，再逐步提 `cruise` 和加长距离。

## 5. 怎么算初值（照公式估，再实测微调）

设起步速 `start`、巡航 `cruise`、加速度 `a`、减速度 `d`，距离单位换成米（mm/1000）：

- **加速段走的距离**（从 start 爬到 cruise）：`s_acc ≈ (cruise² − start²) / (2·a)`
- **减速段刹停距离**（从 cruise 减到 0）：`s_dec ≈ cruise² / (2·d)`
  → 想在 `X` 米内从 cruise 停下：**`d ≥ cruise² / (2·X)`**。这是选 `decel` 的主依据。
- **短程走不到匀速（三角形剖面）**：当目标距离 `target < s_acc + s_dec` 时，
  车根本爬不到 `cruise` 就得开始减速，峰值速度更低——**短距离本来就跑不满速，是正常的**，不是没调好。

> 例：cruise=0.6、d=0.5 → 刹停距离 ≈ 0.36 m = 360 mm。若你的定长任务经常只走 200 mm，
> 那 200 < 360，永远到不了 0.6，峰值只有约 0.32 m/s（三角形顶点）。想短程也快 → 加大 `decel`（和 `accel`）。

## 6. 对症调参表

| 现象 | 多半是 | 怎么调 |
|---|---|---|
| 冲过头 / 停不住 | `decel` 太小 或 惯性打滑 或 `tol` 太大 | 加大 `decel`；必要时降 `cruise`；减小 `tol` |
| 起步卡着不动 / 顿一下才走 | `start` 低于实际启动速 | 加大 `start_mps` |
| 加速太肉、半天不到速 | `accel` 太小 | 加大 `accel` |
| 短距离跑不到设定速度 | 距离 < 加速距离+刹停距离（三角形） | 正常现象；想快就一起加大 `accel`/`decel` |
| 最后几 mm 迟迟不到位、要等超时 | 终段速度衰减到静摩擦以下失速 | 属标定/超时项，**不加纵向下限**（见第 7 节）；或略增 `tol` |
| 走不直、跑偏 | 没开 `heading_hold` 或 `hold_kp` 太小 | 开航向保持、调 `hold_kp` |
| 快到终点时原地打转、不往前 | 终段 `base` 已衰减、航向差速 `corr` 主导了 | 减小 `hold_diff_limit_mps`；见第 7 节 |

## 7. 终段行为提示（务必读，架构评审专门点了）

减速段末尾 `base` 会随剩余距离衰减到 0 附近。这带来两个**已知且预期**的现象：

1. **近终点可能因静摩擦失速**：`base` 掉到车启动速以下时车会停住不动，若此时还没到容差内，
   `dist>=target` 迟迟不满足，最终**靠调用者（Task 层）的超时兜底停**。这与原地定角转 TURN 的做法**同款**——
   不在剖面里加「最小驱动下限」，因为那个值只能实车实测、主机 fake 无静摩擦不可验证。

2. **终段若还有航向偏差，横向差速可能主导**：这时前向 `base` 已经很小、而航向纠偏 `corr` 幅值没变，
   两轮可能呈差速微转、净前进趋缓。**这不是 bug，也不是新的安全问题**（换向过零仍由 `driver/motor` 唯一负责）。
   现场若出现终段打转：减小 `hold_diff_limit_mps`（让终段纠偏别太猛），或靠超时兜底。

> 一句话：**终段的代码停车判据和恒速直行一样，但终段行为不严格一样**——恒速版全程速度不衰减，
> 剖面版终段速度趋 0。后续 Task 层别假设两者终段等价。

## 8. 安全边界（已经做好的，不用你操心）

- **前馈自限 `[0, cruise]`**：剖面输出永远不超过你设的匀速上限，也不出负速——不依赖底盘兜底限幅。
- **未到位 / 退出 / 未初始化 = 安全态**：`Motion_Init` 不发电机命令；到位走 `Chassis_Stop` 转 DONE 静默（刹车保持）；`Motion_Stop` 随时可从正常控制流调用。
- **防堵转看门狗（§8.1，保护 TB6612）**：`profile_stall_ticks`（板上≈0.8s）——**命令在动但编码器不走**（起步速偏低卡住、编码器脱线、撞障）就**快速 `Chassis_Stop`**。这防的正是「堵转时速度环 PWM 顶满、堵转电流灌 TB6612」。是硬件安全项，不是调参项。
- **防过冲看门狗（§8.1）**：`profile_timeout_ticks`（板上≈15s）——移动但迟迟到不了目标（如 `mm_per_pulse` 严重失标）也会兜底停。
- ⚠ **关于烧驱动**：① 速度环**没调过增益时默认 PWM=0，电机不通电**——新板第一次不会堵转，但**车也不会动**（见下方前置）；② 调出增益后，**起步速 `Start` 一定要设到"车真能动起来"的速度以上**（别让它卡住）——这是防堵转的主防线，看门狗只是 0.8s 兜底；③ TB6612 自带热关断是最后一道硬件保险。
- **电机保护在底层**：限速、换向过零死区、命令超时归零全在 `driver/motor`，本原语一行没碰、也没重复做（§8.1）。
- **带载跑车验证归你自理**（§15 裁定）：本手册只保证软件层安全，实际负载/停准行为上板确认。

## 9. 一句话流程

> **进 DEBUG/ProfiledStraight 架空验方向 → 先标 `mm_per_pulse` 让距离测准 → 在 DRIVE 组找起步速 `Start` → 保守设 `Cruise` → 按 `d≥cruise²/(2·刹停距离)` 估 `Decel`、SAVE → 短程低速验四段+停准 → 逐步提速加距离。**（DRIVE 组按钮现场调 + SAVE，不用重编烧录；仅 `mm_per_pulse` 等 s_ms_cfg 占位量要改代码。）

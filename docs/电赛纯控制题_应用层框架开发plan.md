- **硬件约束（硬约束，不可违反）**：近年控制题**强制 TI MSPM0 系列 MCU**（2024、2025 连续两年，且要求裸板备查）；禁履带/麦克纳姆轮；车体约 ≤25×15×25cm；封箱后禁止烧录程序、禁止无线调试。

### 1.3 命题规律（框架设计依据）

1. **赛题=模块重组**：25E = 24H 循迹车 + 23E 云台。2026 大概率继续在"循迹车 ×（云台/视觉/双车协同/避障）"的笛卡尔积里出题。
2. **循迹形态扩展链**：直线→圆角矩形→半圆8字→正方形→含十字/虚线的路网。框架须覆盖"元素识别"，不是单 PID 循线。
3. **时间指标收紧**：单圈 ≤20s、连跑 4 圈计时 → 速度规划（直道加速/入弯减速）是分水岭。
4. **行进间耦合任务出现**（25E 发挥：边走边瞄准、同步画圆）→ 车-云台需要**运动前馈**通道。
5. **赛场约束**：封箱禁烧录 → 按键+OLED 菜单切分问/调参是必备；禁摄像头年份存在 → 纯灰度+IMU 降级方案常备。

## 2. 目标架构（四层）

```
┌──────────────────────────────────────────────────┐
│ L4 应用任务层: 题目状态机（分问切换/任务编排/计时/声光）  │
├──────────────────────────────────────────────────┤
│ L3 应用框架层: 循迹框架 | 云台瞄准框架 | 轨迹执行器      │
│               双机协同协议 | 视觉接入协议                │
├──────────────────────────────────────────────────┤
│ L2 算法组件层: PID库 | 滤波 | 里程计 | 速度规划 | 标定   │
├──────────────────────────────────────────────────┤
│ L1 HAL 抽象层: 同一 API，MSPM0G3507 / STM32 双实现     │
└──────────────────────────────────────────────────┘
```

## 3. 模块开发规格

### P0-A HAL 抽象层【最高优先级】
- 统一 API：`hal_gpio / hal_pwm / hal_encoder(QEI) / hal_uart / hal_timer / hal_adc / hal_oled / hal_key`。
- 双实现：MSPM0G3507（主力）、STM32F4/F1（云台端/备用），编译期 `BOARD_xxx` 宏切换。
- 验收：同一循迹 demo 在两种板上只改宏即可编译运行。

### P0-B PID 库
- 位置式、增量式、串级（外环位置/角度 + 内环速度）三形态统一结构体 `pid_t`。
- 必备特性：积分分离、抗积分饱和、输出限幅、微分先行、死区。
- 验收：单元测试（阶跃响应收敛曲线）+ 在 24H 姿态环、25E 云台两场景复用不改动核心代码。

### P0-C 底盘运动组件
- 编码器测速（M/T 法自适应）、双电机差速同步闭环。
- 里程计（x, y, θ）+ IMU yaw 融合（MPU6050/JY61/HWT101 可插拔）。
- 语义化接口：`chassis_follow_line()`、`chassis_straight(dist_cm, speed)`、`chassis_turn(angle_deg)`、`chassis_arc(radius, angle)`。
- 验收：直线 1m 终点横向偏差 ≤1cm；定角转弯 90°±1°。

### P0-D 赛场生存组件
- 按键+OLED 菜单：选择题目分问、在线调 PID/速度参数、显示实时状态。
- 参数表结构体 + 默认值恢复；`buzzer_beep(n)` / LED 模式封装（题目普遍要求声光提示）。
- 串口日志 + 简易上位机波形输出（赛前调试用）。

### P1-A 循迹应用框架【核心】
四层结构，禁止写成单个循迹函数：
1. **感知层**：N 路灰度 → 归一化 → 加权偏差(-100~+100) → 丢线/全黑/全白标志。
2. **元素识别层**：可注册的元素检测器（特征模式+置信度计数）：直线、直角/圆角弯、十字路口、T 字、圆弧、断线、终点横线、等停标识。新场地元素 = 新增检测器，不改框架。
3. **状态机层**：`TRACK_FOLLOW / GYRO_RUN / ODOM_RUN / TURN_FIXED / SEARCH_LINE / STOP`，元素事件驱动切换。
4. **执行层**：循迹环 PID + 速度规划器（直道加速、入弯减速、梯形速度曲线，曲率前瞻）。
- 验收：24H 场地（半圆+8字）四问全过 + 25E 正方形轨迹 N 圈可设、单圈 ≤18s，代码主体相同仅配置不同。

### P1-B 云台瞄准框架
- **标定模块**：摄像头-激光同轴标定流程（固定像素零点），像素偏差→角度增量映射（线性/查表）。
- **收敛控制**：增量式 PID，连续 N 帧偏差<阈值判定到位 → 触发激光。
- **执行器双后端**：步进（脉冲+方向）/舵机（PWM），统一 `gimbal_set_angle(yaw, pitch)`、`gimbal_move_delta(dyaw, dpitch)`。
- **搜索模式**：目标丢失时螺旋/往返扫描重捕获（对应 25E 基本 3）。
- **轨迹发生器**：复位回原点、沿矩形边走、沿圆弧画（23E、25E 发挥），参数化 `gimbal_track(shape, params)`。
- **运动前馈接口**：`gimbal_feedforward(car_x, car_y, car_theta, car_v)`，行进间瞄准时由车端里程计前馈补偿。
- 验收：静止 2s 内收敛打靶 D1≤2cm；任意朝向 4s 内搜索+命中；行进间画圆同步误差 <1/4 圈。

### P2-A 视觉接入协议
- 通用串口帧：`0xAA 0x55 | msg_id | len | payload | CRC8`。
- 预定义消息：色块坐标、矩形框中心+角点、数字识别结果、光斑坐标、丢目标标志。
- 视觉端模板三份：OpenMV / K210 / K230（二值化、找矩形框、模板匹配）。

### P2-B 双机协同协议
- 主从角色、状态广播（1Hz 心跳）、指令表（启动/暂停/到位/完成/急停）、超时重发、序号防重。
- 覆盖场景：双车协同（21F/22C）、车-云台（25E）。

### P3（余力）
- 超声波/红外测距滤波 + 定距跟随 PID（22C）；绕障/绕柱行为（25K）；泊车状态机（22B）；能量管理策略（21I/22A）。

## 5. 给 AI 的工作指令（执行时按需引用）

- 所有代码遵循上述分层与接口定义；新增功能先判断属于哪一层，不允许跨层调用（L3→L1 必须经 L2）。
- 每交付一个模块附：头文件接口注释、单元测试或场地回归说明、与既有模块的耦合点说明。
- 优先 MSPM0G3507 实现；STM32 实现保持 API 等价。

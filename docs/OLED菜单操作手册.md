# OLED 菜单操作手册（按键 + VOFA 全流程）

> 2026-07-23 生成，对应固件 main 分支（12 调试条目 / 6 参数组）。
> 本文回答一件事：**要调某个东西，先进哪个菜单、VOFA 上看什么发什么、最后把值存在哪**。

## 0. 基础约定（先记住这一节）

**按键**（板载 K1~K4，全程只有这四个键）：

| 键 | 作用 |
|---|---|
| K1 | 上 / 参数值 **+** 一步 |
| K2 | 下 / 参数值 **−** 一步 |
| K3 | 进入 / 确认（在参数上按=进入编辑；在 SAVE 上按=立即存盘） |
| K4 | 返回 / **停止**（运行条目期间唯一有效键，按下=安全停车退出） |

**菜单结构**（两层）：开机进组列表 → 选组 → 组内条目/参数列表。

```
DEBUG  → 12 个运行条目（选中按 K3 开始跑，跑时只认 K4 停）
TUNE   → 循迹外环 LF Kp / Ki / Kd            + SAVE
DRIVE  → Dist mm / Cruise / Start / Accel / Decel + SAVE
CHAS   → 底盘速度环 C Kp / C Ki / C Kd        + SAVE
HEAD   → 航向保持 H Kp/Ki/Kd + 转弯 T Kp + 测试角 Ang + SAVE
SERVO  → 舵机暂存角 S1 deg / S2 deg（无 SAVE，现场标定用）
```

**三条铁规矩**：
1. **运行条目跑着的时候按不了参数**（屏幕锁定 RUNNING，只认 K4）——所以一切调参环都是
   「改参数 → 进条目跑 → K4 停 → 再改 → 再跑」；
2. **任何一个组里按 SAVE = 全部参数一起存**（共享一份 flash 存档，开机自动加载）；
3. 参数显示是 **milli 口径**：屏上 `1200` = 实际 `1.200`（Dist/Ang 例外，直接是 mm/度）。

**VOFA 连接**（需要看波形的流程用）：
- 接线：主控 PA1(TX) → USB 串口板 RX，PA0(RX) ← TX，共地；
- 串口：**230400，8N1**；VOFA+ 协议选 **JustFloat**；
- 发命令格式：命令输入框输入 `名字=数值` 回车（也支持 `名字:数值`），如 `LP=150`。

## 0.5 参数单位与推荐起手值（让你有方向，不是标定终值）

> ⚠️ 下表「起手值/搜索范围」是**量级参考**（按输出满格 1000 PWM、10ms 控制周期的
> 物理账推的），你的车实测为准——它的作用是让你第一把不至于差一千倍，不是免调。

| 组·参数 | 物理单位 | 含义 | 屏显换算 | 起手值(屏显) | 搜索范围(真实值) |
|---|---|---|---|---|---|
| CHAS·C Kp | PWM/(m/s) | 每 1 m/s 速度误差立即给多少 PWM（满格 1000） | ×1000 | `100000`(=100) | 50 ~ 600 |
| CHAS·C Ki | PWM/(m/s)/拍 | 误差不消时每 10ms 再追加多少 PWM（消稳态差主力） | ×1000 | `100000`(=100) | 50 ~ 500 |
| CHAS·C Kd | — | **保持 0**（速度环不加 D，测速量化会抖） | ×1000 | `0` | 0 |
| TUNE·LF Kp | (m/s)/mm | 偏线 1mm 给多少差速纠偏 | ×1000 | `10`(=0.010) | 0.005 ~ 0.05 |
| TUNE·LF Kd | (m/s)/(mm/拍) | 压蛇形阻尼 | ×1000 | `0` 起，蛇形再加到 5~20 | 0 ~ 0.03 |
| TUNE·LF Ki | — | 一般保持 0（弯道积分会兜圈） | ×1000 | `0` | 0 |
| DRIVE·Cruise | m/s | 定长直行巡航速度 | ×1000 | `200`(=0.2) | 0.15 ~ 0.5 |
| DRIVE·Start | m/s | 起步速度（低于电机启动速度会趴窝） | ×1000 | `80`(=0.08) | 0.05 ~ 0.15 |
| DRIVE·Accel/Decel | m/s² | 加/减速度（满载不过第一个降它） | ×1000 | `300`(=0.3) | 0.2 ~ 0.8 |
| DRIVE·Dist mm | mm | 直行距离 | 直读 | `1000` | — |
| HEAD·H Kp | (m/s)/° | 直行偏航 1° 给多少差速拉回 | ×1000 | `10`(=0.01) | 0.005 ~ 0.05 |
| HEAD·H Ki/Kd | — | 先都 0（Kd 抖再说，Ki 基本不用） | ×1000 | `0` | 0 ~ 0.01 |
| HEAD·T Kp | (m/s)/° | 原地转剩 1° 时还给多少速度 | ×1000 | `20`(=0.02) | 0.01 ~ 0.06 |
| HEAD·Ang | 度 | TurnTest 测试角 | 直读 | `90` | 90/180 |
| SERVO·S1/S2 | 度 | 舵机角，90=中位，0~180 | 直读 | `90` | 机构限位内 |

**按键步长**（每按一次 K1/K2 变多少，屏显口径；按住不放**没有连发**，一步一按）：

| 组 | 步长 | 设计账（量程 ÷ 步长 ≈ 需要按几下） |
|---|---|---|
| CHAS Kp/Ki | `10000`(=10.0) | 量程 50~600 → 起手 100 只按 10 下 |
| CHAS Kd | `1000`(=1.0) | 基本不用（保持 0） |
| TUNE LF 三项 | `2`(=0.002) | 量程 5~50 → 精调一格 ≈ 典型值的 1/5 |
| DRIVE Cruise | `20`(=0.02m/s) | 0.2→0.5 共 15 下 |
| DRIVE Start | `10` | 起步阈值附近要细 |
| DRIVE Accel/Decel | `50`(=0.05) | 0.2→0.8 共 12 下 |
| DRIVE Dist | `50`mm | 1m→2m 共 20 下 |
| HEAD H 三项 | `2` | 与 LF 同量级同精度 |
| HEAD T Kp | `5`(=0.005) | 量程 10~60 → 10 下内 |
| HEAD Ang | `15`° | 90/180 都整除 |
| SERVO S1/S2 | `5`° | 全程 0~180 共 36 下，机构标定够用 |

**两条保护你要知道**：
1. **增益按不成负数**：所有增益（LF/CHAS/HEAD 共 10 个）K2 按到 0 就停在 0——负增益
   等于反向纠偏/正反馈飞车，固件直接拒绝（Ang 例外：负角=反方向转，是合法用法）；
2. **量级自检口径**：底盘增益是「百量级」（屏显十万量级），循迹/航向增益是「百分之几」
   （屏显个位~两位数）——你按出来的数和这个量级差 10 倍以上，先停下来对一遍单位。

## 1. 调速度环（唯一必须用 VOFA 看波形的调参）

**路径：DEBUG → SpeedTune（VOFA 调）→ 终值抄进 CHAS 组 → SAVE**

① 车架空（轮子悬空），连好 VOFA，进 `DEBUG → SpeedTune`。

② VOFA 波形通道（JustFloat，10 通道）：

| ch | 内容 | ch | 内容 |
|---|---|---|---|
| 0 | 左轮 Kp | 5 | 右轮 Kd |
| 1 | 左轮 Ki | 6 | 左轮目标速度 m/s |
| 2 | 左轮 Kd | 7 | 右轮目标速度 m/s |
| 3 | 右轮 Kp | **8** | **左轮实测速度**（主看） |
| 4 | 右轮 Ki | **9** | **右轮实测速度**（主看） |

③ VOFA 命令（发了**立即生效**，边看边调）：

| 命令 | 作用 | 命令 | 作用 |
|---|---|---|---|
| `LM=0.3` | 左轮目标 0.3 m/s（给阶跃用） | `RM=0.3` | 右轮目标 |
| `LP=150` | 左轮 Kp（**注意量级：百位数**，见 §0.5） | `RP=150` | 右轮 Kp |
| `LI=100` | 左轮 Ki | `RI=100` | 右轮 Ki |
| `LD=0`   | 左轮 Kd（**速度环不加 D**，保持 0） | `RD=0` | 右轮 Kd |

④ 调法：先发 `LP=150` `RP=150` `LI=100` `RI=100` 起手 → 发 `LM=0.3` `RM=0.3` 给阶跃 →
看 ch8/ch9 跟 ch6/ch7 的贴合——跟不上/有稳态差加 `LI/RI`，来回振荡减 `LP/RP`；
调完发 `LM=0` `RM=0` 停轮。

⑤ **存档（关键一步）**：K4 退出 SpeedTune → 进 `CHAS` 组 → 把终值按出来
（比如 Kp 定在 180，就把 `C Kp` 按到 `180000`——步长 10000/次，18 下；屏显=真实×1000）
→ 光标到 `SAVE` 按 K3。
⑥ 验证：整机断电重启 → 再进 SpeedTune → 波形 ch0~5 显示的就是存档值。

> 为什么要抄一遍：VOFA 里改的增益只活在内存，断电即失；CHAS+SAVE 才是进 flash。

## 2. 调循迹（按钮调，看车不看波形）

**路径：TUNE 组改增益 ↔ DEBUG → LineFollow 试跑，循环**

① 前提：速度环已按 §1 调好存档；灰度已在 GrayTest 里标定过。
② `TUNE` 组：`LF Kp`（主力，先调）、`LF Kd`（压蛇形，其次）、`LF Ki`（一般保持 0）。
③ 循环：改一个值 → `DEBUG → LineFollow` 上线跑 → 看姿态 → K4 停 → 回 TUNE 再改。
   直道蛇形=Kp 大或 Kd 小；弯道外甩丢线=Kp 小。
④ 满意后 `SAVE`。
> 循迹基础速度当前是代码里的保守常量 0.25 m/s（不在菜单里）；要提速找我改一行重烧。

## 3. 调定长直行 / 标定距离（DRIVE 组）

**路径：DRIVE 组设参数 → DEBUG → ProfiledStraight 跑一段 → 卷尺量**

① `DRIVE` 组：`Dist mm`（要跑的距离，默认 1000）、`Cruise/Start`（巡航/起步速度，milli m/s）、
   `Accel/Decel`（加减速度，milli m/s²）。
② 进 `DEBUG → ProfiledStraight`：车按梯形速度剖面跑 Dist 毫米自动停。
③ **标定 mm_per_pulse**：命 1000mm，卷尺量实跑 L mm——把 1000 和 L 告诉我，我改标定值重烧
   （新值 = 旧值 × 1000 / L）；复测到停点误差 ≤±20mm。
④ 满意后 `SAVE`（剖面参数和 Dist 都会存）。

---

## 4. 调转弯与航向（HEAD 组 + TurnTest）

**路径：HEAD 组改增益 ↔ DEBUG → TurnTest 转一把，循环**

① 前提：IMU 在 ImuTest 里验过（valid=1、漂移合格）；速度环已存档。
② `HEAD` 组五个参数：

| 项 | 管什么 | 起手值建议 |
|---|---|---|
| `T Kp` | **原地定角转**的力度（TurnTest 主调这个） | 从 5 起往上加 |
| `Ang`  | TurnTest 每次转多少度 | 90 |
| `H Kp/Ki/Kd` | **直行时的航向保持**（ProfiledStraight 用） | Kp 从 10 起，Ki/Kd 先 0 |

③ 循环：改 `T Kp` → 进 `TurnTest`（车原地转 Ang 度自动停，最长 10 秒强制停）→
   量角器量实际转角 → K4 → 再改。转不到位加 T Kp，过冲/来回摆减 T Kp。合格线 90°±2°。
   **第一次进如果转反了**：是 IMU 极性，告诉我改 heading_sign，别调参数硬掰。
④ 直行航向保持：调 `H Kp` 后跑 ProfiledStraight 看 1m 侧偏（≤±50mm 合格）。
⑤ 满意后 `SAVE`。

## 5. 舵机摆位（SERVO 组 + ServoTest）

**路径：SERVO 组设角 → DEBUG → ServoTest 施加 → K4 释放，循环**

① 空转态（没跑任何条目时）在 `SERVO` 组把 `S1 deg` 设到安全位（如 90）；
② 进 `DEBUG → ServoTest`：舵机立刻摆到暂存角并保持；
③ K4 退出=释放（舵机可手拨）；回 SERVO 改角再进，直到找到机构的开/合角，**把两个角度记在纸上**
   （此组不进 flash，赛题定型后我把终值固化进代码）。

## 6. 诊断页面详解（接上外设看数据，不调参）

每条按同一格式：**它在干什么 → 你要做什么 → 什么算好 → 什么算坏**。

### 6.1 MotorDir（电机方向自检，肉眼）

- **在干什么**：进页后固件自动让两个轮子一起以 20% 功率转 2 秒、反转 2 秒，无限循环。
- **你做什么**：车架空，盯着轮子，拿纸记：「命令正转时，左轮实际朝哪转、右轮朝哪转」。
- **算好**：两轮同步换向、节奏整齐、K4 一按立刻停。
- **算坏**：某轮不转（查 TB6612 接线/供电）；两轮方向相反或都反（**正常现象**，把记录发我，
  固件改一个宏就正，不用重新接线）。

### 6.2 EncoderTest（编码器极性/测速，VOFA）

- **在干什么**：固件每 10ms 读一次两个编码器，把「累计脉冲数」和「换算出的轮速」发给 VOFA。
  它自己不驱动电机——轮子要你用手拨。
- **你做什么**：**用手朝车头前进方向**拨左轮，再拨右轮。
- **看什么**：ch0=左轮累计脉冲、ch1=右轮、ch2=左轮速度 m/s、ch3=右轮。
- **算好**：拨哪个轮哪路动；**前进方向拨动时数值变正**；停手后脉冲数保持不动（不漂）。
- **算坏**：变负=AB 两相接反（**预期内事故**，告诉我改一处符号，别动线）；
  不动=线没接对（查 U33/U34 对应 PB10/11、PB7/9）；乱跳=接触不良。

### 6.3 GrayTest（灰度标定助手，只看 OLED，不用电脑）

- **在干什么**：每 10ms 读 12 路灰度的黑白判断，OLED 整屏画四行面板：
  第 1 行**实时值**（12 个 0/1）、第 2 行**粘滞值**（触发过就保持亮，抓瞬断）、
  第 3 行**跳变计数**（哪路在抖）、第 4 行十六进制位图。
- **你做什么**：拿黑纸条从探头阵列一端划到另一端；再调传感器板上的电位器直到黑白分明。
- **算好**：黑纸划到哪一位哪一位翻转，顺序连续；白地上全灭、黑线上对应位亮。
- **算坏**：某位常亮/常灭=该路坏或电位器没调好；跳变计数疯涨=阈值卡在临界（拧电位器）。
- **顺手必做**：黑纸只碰**车头最左**那个探头，记下屏上亮的是第几位——发我（定软件左右）。

### 6.4 ImuTest（IMU 健康检查，VOFA）

- **在干什么**：每 10ms 收 IMU 串口数据，把姿态和「通信健康度」发 VOFA；同时从进页起
  持续统计「航向角自己在飘多快」（静置漂移）。
- **你做什么**：模块**放桌上别动**，进页等 60 秒；然后拿起来水平转 90° 再看。
- **看什么**：ch0=航向角° ch1=角速度°/s ch3=valid(1=通着) **ch4=漂移°/s**
  ch5=收到帧数 ch6=校验错 ch7=丢字节。
- **算好**：ch3=1；ch5 匀速涨；ch6/ch7 一直是 0；静置 60s 后 **|ch4|≤0.05**；
  手转 90° 时 ch0 变化 ≈90。
- **算坏**：ch3=0=没通（查 PA25/26 和供电）；ch6 涨=波特率错；ch4 大=器件飘
  （先查供电纹波和桌子震动，再怀疑器件）。

### 6.5 BeaconTest（声光验收，人耳人眼，不用电脑）

- **在干什么**：进页后固件自动按固定顺序轮播 7 种提示模式，每种 2 秒，循环播放：
  ①短响 0.1s → ②哔-哔双响 → ③长响 0.5s → ④LED 常亮 → ⑤LED 慢闪(1s 周期) →
  ⑥LED 快闪(0.2s 周期) → ⑦声光同步闪。这 7 种就是比赛里「到点提示/完成确认/告警」
  会用的全部语义。
- **你做什么**：站旁边听和看一整圈（14 秒），对着上面的顺序数。
- **算好**：7 种都能分辨、顺序对得上；K4 一按立即全静（这是安全验收点）。
- **算坏**：全程不响=蜂鸣器没接/坏（查 PB18 与 H4 冲突）；一直长鸣不停=蜂鸣器是
  无源型或低电平触发（告诉我，固件换个驱动方式，接线不动）；灯不亮=PB22 的 LED 问题。

### 6.6 ServoTest（舵机摆位，肉眼）

- **在干什么**：进页后把你在 SERVO 组预设的角度**施加**给舵机（进页前舵机是松的）；
  页面里每 10ms 按限速斜坡驱动舵机保持/走向目标角。
- **你做什么**：见 §5 的「改角→进页看→K4 释放→再改」循环。
- **算好**：进页舵机走到预设角并稳住不抖；K4 后能用手拨动（=已释放）。
- **算坏**：不动=信号线（PA27/PB1）或**独立 5V 供电**没接；抖动/复位=舵机从主控板取电了
  （必须独立供电只共地）；到端点嗡嗡响=顶到机械限位（记下角度发我设软限位）。

### 6.7 LinkTest（ESP32-C3 无线链路，VOFA）

- **背景**：两台车的通信 = 各自的主控接一块 ESP32-C3 小板（串口相连），两块 ESP32-C3
  之间用 ESP-NOW 无线互传。对主控来说就是「串口发出去、串口收回来」。
- **在干什么**：进页后固件每 200ms 从串口发一个「心跳包」（我还活着），同时不停收听；
  600ms 内收到过任何有效包就判定「链路活着」(alive=1)。协议 v2 起每个包带序号：
  普通包（状态/心跳）丢了会体现在「测得丢包数」里；「事件包」（指令类）收不到确认
  会自动重发（40ms 一次、最多 8 次），所以**指令要么送达、要么明确报失败**，不会悄悄丢。
- **看什么**：ch0=alive ch1=累计收到的包 ch2=校验错的包 ch3=**测得丢包数**（对方发了
  但没到的普通包数量——无线质量直读） ch4=事件重发次数 ch5=事件送达数 ch6=事件放弃数
  （重发 8 次仍无确认） ch7=已发心跳数 ch8=丢字节 ch9=**port_absent**（1=串口引脚还
  没定案，固件如实报「没接」）。
- **现在（引脚未定案）**：进页看到 ch9=1、其他全 0——**这是正常的**，说明固件在位等引脚。
- **引脚定案后第一步（自环回）**：先不接 ESP32，把这个串口自己的 TX 和 RX 用一根杜邦
  短接——自己发的心跳自己收到 → ch0 变 1、ch1 每秒 +5。这一根线就证明了主控这半边全通。
- **接上 ESP32-C3 双机**：两台车各进 LinkTest → 两边 ch0 都=1（收到的是**对方**的心跳）、
  ch1 持续涨、ch2 不涨。把一台断电 → 另一台 0.6 秒内 ch0 掉 0（这就是将来「跟车断联
  自动停」的判断依据）。
- **算坏**：自环回都不通=串口配置/接线问题；双机 ch2 猛涨=ESP32 透传固件波特率不匹配。

### 6.8 VisionLink（视觉通信联调，VOFA；没有摄像头也能测）

- **背景**：视觉组的香橙派识别出目标后，通过串口把「坐标包/识别结果包」发给主控；
  主控开机时要先发「选题包」告诉视觉跑哪道题，视觉回显同样内容表示确认（握手）。
- **在干什么**：进页后固件立刻发一次选题包，**没收到确认就每 0.5 秒自动重发**；
  同时把收到的坐标、识别结果、通信计数全发 VOFA。
- **看什么**：ch0/ch1=目标坐标 x/y ch2=坐标包序号（涨=在持续收）ch3/ch4=识别结果两个
  字节 ch5=结果包序号 **ch6=握手确认(1=视觉已应答)** ch7=选题包重发次数。
- **没有视觉硬件时**：用电脑 USB 串口杜邦接 PA8/PA9，串口工具照 §（杜邦清单 8）的帧格式
  发包代打——固件分不出对面是香橙派还是电脑。
- **算好**：对端回显选题包后 ch6 变 1 且 ch7 停涨；打坐标流时 ch2 匀速涨；
  故意发半截乱码再发好包，ch2 能继续涨（自同步恢复能力）。
- **算坏**：ch7 一直涨=对端没收到/没回（查 TX/RX 是否要对调，见确认清单 H2）；
  ch2 不涨但对端在发=波特率（应 230400）或帧格式不对。

### 6.9 LineFollow / ProfiledStraight / TurnTest（整车运行条目）

这三个不是诊断页，是真跑车的：分别见 §2（循迹）、§3（定长直行）、§4（转弯）。
共同点：进页即开跑、**K4=唯一的刹车键**、都有超时自动停底线（直行 15s / 转弯 10s）。

## 7. 赛日一分钟自检（每次上电）

进 CHAS 看 `C Kp` 不是 0（存档在）→ 进 DEBUG 随便进退一个诊断页（按键/OLED 活着）→ 开跑。
若 `C Kp`=0：存档丢了，按 §1⑤ 从纸质记录重录入 SAVE（这就是为什么每次调完要抄纸）。

# 12 路灰度传感器配置指南（NCHD1「迹」阵列）

面向：调车的人。本文只讲**你要动手做什么**，固件侧已就绪。

器件：武汉无名创新 **NCHD1（迹）**，12 路线性阵列（NC-HD12 扩展板）。
对应固件：`hc-team/driver/gray/`（P9.T1，2026-07-17）。

> **本文是器件事实的唯一存档。** 原厂手册 PDF 与 STM32 示例工程已于 2026-07-17
> 按用户要求删除，且它们从未进过 git —— **删掉就再也拿不回来了**。

## 0. 一句话

**深色（黑线）→ 输出高电平 1；浅色（白底）→ 输出低电平 0。**
固件读到的 `bit i = 1` 就是「第 i+1 路下面是黑的」。仅此而已，没有别的语义。

---

## 1. ★ 必须先确认的一件事：供电电压（唯一硬件风险项）

| 情况 | 后果 |
|---|---|
| 模组用 **5V** 供电 | 手册 p.31 明写：**输出高电平约 4.0V** |
| MSPM0G3519 的 PB 脚不耐 5V | 4.0V 灌进去 = **烧 IO** |

厂商手册 p.31 原话（转录）：

> 对于部分不支持 5V 电平输入的单片机，保险起见可以将模块输出信号串联一个
> 300-1000 欧姆的电阻后再接入单片 IO 口，或者直接使用 3.3V 给灰度传感器模块供电更为稳妥。

**动手之前必须二选一：**

1. **用 3.3V 给模组供电**（最稳妥，推荐）。代价：手册 p.21 说 5V 供电能增加有效检测距离，
   3.3V 下探测距离会缩水 —— 但推荐安装高度只有 10–30mm，本来就用不满。
2. 若坚持 5V：**先查 MSPM0G3519 数据手册，确认那 12 个 PB 脚是否耐 5V**；不耐就每路串 300–1000Ω。

> 我没有核实 MSPM0G3519 的 IO 耐压 —— 那是器件手册里的硬件事实，
> 固件层无法自证，也不该由固件层替你拍板。**这一条卡在你这里。**

## 2. 电位器调节（12 路每一路都要调）

手册 p.23 的三步法，逐路做：

1. 固定好安装高度（见 §3），把电位器**顺时针拧到底**；
2. 传感器对着**浅色**背景，**逆时针**缓慢拧，直到绿色 OUT 灯**亮起**，
   然后**继续逆时针再拧一点点**（这一点点是故意留的电压裕度，防止车抖动/地面不平造成误判）；
3. 移到**深色**背景，OUT 灯应**由亮变灭**。在深色上把传感器上下微抬，灯不跳变 = 这一路调好了。

手册 p.24 的口诀（原文转录）：

```
高度固定顺拧到底
浅色逆拧待灯亮起
求稳继续微拧少许
移至深色绿灯即灭
上下微抖灯仍不变
反复测试无惧颠簸
```

> 灯的含义别记反：**灯灭 = 深色 = 输出 1**；灯亮 = 浅色 = 输出 0（手册 p.8）。

## 4. ★ 位序：必须实测，不许猜

**这是本文最重要的一条。**

存在一个**已知矛盾**，固件无法自行裁决：

| 来源 | 说法 |
|---|---|
| 厂商手册 p.34 | `bit1..bit16` = 阵列**最右端 → 最左端**，即 **P1 = 最右** |
| 本仓 `board.syscfg` 注释 | 「12 路灰度，物理排列**从左到右** IN1 -> IN12」，即 **IN1 = 最左** |

两者不可能同时为真。而**哪个物理位置的传感器接到了 `PIN_IN1`（PB27），
是你们插线时决定的硬件事实** —— 固件读到的只是「PB27 这个脚是高还是低」，
它没有任何办法知道那个脚连到车头左边还是右边。

### 怎么测（一分钟）

1. 车放在黑线上，把黑线**移到阵列最左端**（或把车往右挪，让最左那路压住线）。
2. 读 `Gray_ReadDarkBitmap()` 的返回值。
3. - 若 **bit0** 置位 → IN1 = 最左，本仓注释对，厂商约定不适用于你们的接法。
   - 若 **bit11** 置位 → IN1 = 最右，厂商约定对，本仓注释错（届时请改掉那条注释）。

### 测出来以后

把结论告诉我，或者直接写进将来的循迹权重表。

> ⚠ **修正点只能有一个**，且必须在**上层的权重表**里。
> **绝对不要**在 `driver/gray/` 里加一个「反转开关」——
> 那会和权重表的方向互相抵消，两处都对的时候车是对的，
> 改了任何一处车就反着走，而且极难查。
> 这正是 `encoder.c` 的 `s_direction_sign[]` 立下的规矩（`AGENTS.md` §8.2 单一数据所有者）。

## 5. 固件侧已经做了什么 / 不做什么

驱动只有一个函数：

```c
uint16_t Gray_ReadDarkBitmap(void);   // bit i = PIN_IN(i+1) 当前为深色
```

**已做**：12 路**一次原子读取**（全 12 路在同一端口 PORTB，一次 `DL_GPIO_readPins` 读全，
路间零时间偏斜）。这是 `board.syscfg` 特意让 IN4 占 PB8、而不是跨端口的 PA7 换来的性质。

**刻意不做**（都不是遗漏，各有理由）：

| 不做 | 为什么 |
|---|---|
| 去抖 / 滤波 | 器件侧已有 C2(100pF) 滤波 + 电位器留的比较迟滞裕度（§2 第 2 步）。固件再滤一次 = 两个所有者 |
| 丢线判断 | 「全 0 = 丢线」是循迹算法对位图的解读，不是器件事实 |
| 误差量化 | 厂商示例那张 -11..+11 的映射表是**循迹算法**，归 Middleware，不归 Driver |
| 赛道特征识别 | 起始线/十字/直角同理 |
| 初始化函数 | 器件要求的唯一初始化动作是「IO 配成输入」（p.21/p.32），SysConfig 已经做了 |

> 厂商示例里那张偏差映射表（12 路 → -11..+11）我没有丢，它的设计前提记在这里：
> **10mm 线宽下，正常最多 2 路同时压线；不丢线时至少 1 路为高**（手册 p.38）。
> 将来写循迹算法时，这是权重表的出发点。

## 6. 接线（本仓实际）

12 路全部接在 **PORTB**（`board.syscfg` 的 `GPIO_LINE_SENSOR` 组）：

| 通道 | 引脚 | | 通道 | 引脚 |
|---|---|---|---|---|
| IN1 | PB27 | | IN7 | PB17 |
| IN2 | PB12 | | IN8 | PB19 |
| IN3 | PB13 | | IN9 | PB21 |
| IN4 | **PB8** | | IN10 | PB14 |
| IN5 | PB20 | | IN11 | PB0 |
| IN6 | PB26 | | IN12 | PB24 |

> **12 路必须全部留在 PORTB。** 跨端口会让 SysConfig 不再生成 `GPIO_LINE_SENSOR_PORT`
> 组级宏（`gray_hw.c` 会编译失败 —— 这是刻意的），并且 12 路再也无法一次读完。
> IN4 用 PB8 而不是引脚表建议的 PA7，就是为了这个。

## 7. 常见问题（手册 p.31–p.32 转录）

**Q：接上单片机后，不管深色浅色，OUT 灯都常亮/常灭？**
A：把对应 IO 配成**输入**再接。否则单片机 IO 的随机或输出电平会反过来影响模块输出。
（本仓 SysConfig 已配为 INPUT，若出现此现象请查接线而非固件。）

**Q：数字还是模拟？**
A：模组出厂即数字（DEF 处焊有零欧电阻 R6，等效短接 OUT-IO）。本仓 12 脚接的是普通 GPIO
而非 ADC 通道，**只能用数字**。要改模拟得拆 R6 并改焊 OUT-AD 焊盘 —— 那会同时需要改硬件和固件，
不在当前设计内。

**Q：为什么用灰度不用普通红外？**
A：手册 p.2：灰度用高亮白色聚光 LED + 抗干扰光电管，抗黑暗/普通灯照射环境，抗干扰能力更强。

## 9. 现场标定流程（配合固件 GrayTest 标定助手，2026-07-21 W7）

先把一件事说死：**阈值旋钮就是电位器，固件里没有阈值**（§2/§7；全手册无软件阈值、
无一键标定、无串口配置——2026-07-21 PDF 40 页全文复核）。光线变了 → 重拧电位器，
没有别的路。固件能做的是把「拧完对不对」的**校验**从盯 12 个小灯变成 OLED 一屏读数。

### 9.1 标定助手面板怎么读

菜单 → DEBUG → **GrayTest** 进入。OLED 四行（左端 = bit0 = G1 = PB27，
只是通道序，**不代表车上左右**——左右要按 §4 实测）：

```
L:#.#..#.....#     实时位图：# = 该路此刻读到深色
S:#.#..#.....#     粘滞位图：进页以来曾经深色过的路（OR 累积）
T:...2........     跳变计数：该路电平翻转次数（.=0，1..9，*≥10）
X:825 N:04         位图十六进制（bit0=最低位）+ 当前深色路数
```

- 面板显示的是 **MCU 实际读到的电平**（含接线和串阻链路）。模组上的 LED 只反映模组侧：
  灯对而屏不对 = 接线/串阻问题，不是电位器问题。
- **清零统计 = 退出再进入**（BACK → 再 ENTER）。没有单独的清零键。
- 接了 PC 时 VOFA 的 G1..G12 同步可用，两面显示同一份数据。

### 9.2 初始化（首次装车 / 换车 / 换场地的全量标定）

1. 固定安装高度 10–30mm（§3），确认供电方案（§1）。
2. 逐路按 §2 口诀拧电位器（顺拧到底 → 浅色逆拧待灯亮 → 再拧少许留裕度 → 深色验灭）。
3. 进 GrayTest，**白底校验**：车放纯白底面静置 5 秒再轻压车身晃几下——
   `L` 应全 `.`、`S` 应全 `.`、`T` 应不增长。哪路出 `#` 或计数，只重拧那一路。
4. **压线校验**：车压黑线慢慢滚过全阵列——压线路应实心 `#` 且 `T` 不涨；
   `N` 走到每个位置都应是 1–2（10mm 线宽最多 2 路同时压线，手册 p.38）。
5. **位序实测**（§4）：黑线压到阵列最左端，看 `X` 行——`001` 则 bit0=最左，
   `800` 则 bit0=最右。结论回报，写进循迹权重表（唯一修正点）。

### 9.3 现场快速复检（比赛场地光线变化后，1–2 分钟）

1. 进 GrayTest，车放赛道**白色区**：`L`/`S` 应全 `.`、`T` 不增长。
   有 `#` 的路 = 现场光线下误判深色 → 该路电位器**逆时针**微拧（升比较电压裕度），
   拧到 `S`/`T` 清零重测（BACK→ENTER）后保持干净。
2. 车压赛道黑线静置、再上下轻抖：压线路 `L` 应稳 `#`、`T` 应不涨。
   跳变的路 = 比较电压离深色电平太近 → 该路**顺时针**微拧少许（降低比较电压）后重测；
   若白底、黑线两头都不稳，别来回猜，该路直接按 §2 三步法重调一遍。
3. 两步都干净即完成。**只动出问题的那几路**，别全阵列重来。

> 原理备忘（手册 p.16/p.23）：电位器逆时针 → 比较电压 V-com 升高 → 更不容易判深色。
> 现场光线强弱本质是改变了反射进光敏管的光量，电位器就是在新光量下重新划黑白分界线。

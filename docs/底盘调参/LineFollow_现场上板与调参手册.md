# LineFollow 循迹运行条目 — 现场上板与调参手册

> 对应代码：`hc-team/app/system/app_compose.c`（DEBUG 组第 5 条 `LineFollow`）+ W5 TUNE 参数组。
> 契约：`agent/phase4_app_rewrite/plan_app_first_order.md` §26（W6）。代码提交 `b907003`。
> 本手册讲**怎么用、第一次上板注意什么**，不讲代码内部实现。

---

## 1. 这是什么

- **DEBUG / LineFollow**：一条运行条目。进去后小车开始沿黑线跑（循迹），退出后确定性停车。
- **TUNE 组**：改循迹外环 PID 三个增益 `LF Kp / LF Ki / LF Kd`，`SAVE` 一次写进片内 flash（掉电不丢）。

两者配合 = 一套现场调参闭环：**选它跑 → 退出 → 改增益存盘 → 再选它跑 → 看效果**。
不用 VOFA、不用改代码重新烧录。

---

## 2. 菜单操作（固定按键映射）

| 按键 | 作用 |
|---|---|
| K1 | 上移 / 数值 **加** |
| K2 | 下移 / 数值 **减** |
| K3 | 进入 / **确认**（在 SAVE 项上 = 存盘） |
| K4 | 返回 / **退出当前运行条目**（退出即安全停车） |

- 跑循迹：进 `DEBUG` 组 → 选 `LineFollow` → K3 进入，小车开始跑。K4 退出即停。
- 调增益：进 `TUNE` 组 → 选 `LF Kp`（或 Ki/Kd）→ K3 进编辑 → K1/K2 加减 → K4 回列表 → 选 `SAVE` → K3 存盘。
  - **编辑只改内存，存盘要点 SAVE**。不点 SAVE 就退，掉电会丢。

## 3. ⚠ 第一次上板，按这个顺序来

### 步骤 0：先确认灰度方向（不是在这个任务里改）

小车「看到偏左往哪修」取决于灰度位序的左右。**这个左右以 DEBUG 组的 `GrayTest` 实测为准**：
用 GrayTest 看哪个物理探头触发哪一路，读出车头左右对应位图哪一端。

- 现在代码里填的方向是**猜的占位值**，等灰度实测出来后会在一个公共层统一定，两个任务共用（已登记为待决项 Q12）。
- **在灰度方向定案前，别急着放地上全速跑**——方向猜反的话小车不会往回修，会直接冲出赛道。

### 步骤 1：架空 / 低速验证转向

把小车**架空**（轮子离地）或用手拦着，进 `LineFollow`，手动把探头在黑线上左右移动，看轮子的反应方向对不对：

- 探头偏到线左边时，应当往**让它回到线上**的方向修，而不是越修越偏。
- 方向反了 → 先解决步骤 0 的灰度方向，别在这个任务里硬掰。

### 步骤 2：第一次跑可能「直愣愣不拐弯」——这是正常的

出厂默认 `Kp/Ki/Kd = 0`，意思是「看到偏了也不动方向」。所以第一次进 LineFollow，小车很可能只会往前直冲、不循迹。

**这不是坏了。** 去 `TUNE` 组把 `LF Kp` 加上去、`SAVE`，再回来跑就有循迹反应了。
调参口诀：**先只加 Kp 到能跟线**（可能有点抖/画龙）→ 再加一点 `Kd` 压抖 → `Ki` 一般最后、少量或不加。

### 步骤 3：逐步放开

方向对了、增益调到能跟线后，再从低速逐步往上提。速度相关的标定值见下一节。

## 4. 占位标定值 — 硬件到了要替换

当前配置 `s_lf_cfg`（在 `app_compose.c`）是**保守的占位值**，带 `UNCALIBRATED` 注释的都要上板按实测替换：

| 字段 | 当前占位 | 含义 / 替换依据 |
|---|---|---|
| `pitch_mm` | 12.0 | 相邻探头中心间距（mm），量一下实际探头板 |
| `bit0_is_left` | true（**猜**） | 位图 bit0 是不是车头左端 → **以 GrayTest 实测为准**（Q12） |
| `straight_speed_mps` | 0.25 | 直道巡航基速上限（m/s），保守低速起步，稳了再提 |
| `min_speed_mps` | 0.12 | 入弯最低基速，必须 ≤ 直道速 |
| `curve_error_mm` | 20.0 | 误差多大时降到最低速（入弯减速灵敏度） |
| `accel/decel_mps_per_s` | 0.5 / 1.0 | 提速/降速斜坡快慢 |
| `diff_limit_mps` | 0.30 | 差速修正上限，越大转向越猛 |
| `recovery_error_mm` | 32.4 | 丢线后往记忆方向回退的力度（≈2.7×pitch） |
| `lost_timeout_ms` | 1500 | 丢线找不回来多久后停车（安全项，有界） |
| `element_enable_mask` | 0 | 循迹元素（断线/横线/岔路）检测开关，**先关**，只跑基本循迹 |

> **PID 增益不在这张表里** —— 增益走 TUNE 组按钮调、param_tune 存 flash，不在 `s_lf_cfg`。
> 所以改上面这些占位值要重新编译烧录；改增益不用，现场按钮就行。

## 5. 安全边界（已经做好的，不用你操心）

- **退出必停**：K4 退出 → `LineFollow_Stop` → 底盘刹停。空转态小车不会自己动。
- **配置无效不动**：万一 `pitch_mm` 或 `diff_limit` 填成 0，进入后小车保持不动（不会乱冲），不是死机。
- **电机保护在底层**：限速、换向过零死区、命令超时归零这些保命逻辑都在 `driver/motor`，本条目一行没碰、也没重复做。
- **带电跑车验证归你自理**（§15 裁定）：本手册只保证软件编排安全，实际负载行为上板确认。

## 6. 一句话流程

> **架空验方向（灰度定）→ TUNE 把 Kp 调起来 SAVE → 放地上低速跑 → 边跑边 TUNE 微调 → 逐步提速。**

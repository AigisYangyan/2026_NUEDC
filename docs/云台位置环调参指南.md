# 云台位置环静态调参指南（GimbalTune，W8）

面向：现场操作者（无需读代码）。对象：云台（二维平台）视觉瞄准位置环——
`vision_aim` 位置式 PD（kp/kd）+ 死区（DB）+ 每拍步长限幅（MS）。
方法：菜单进 `DEBUG → GimbalTune` 条目，用 VOFA+ 上位机发命令改参数、看波形定型；
**调好的终值回填代码里的 `s_gt_cfg`（`app_compose.c`）编译期默认，不进 flash**。

> 与 TUNE/DRIVE 按钮调参组的分工：TUNE 组管比赛日现场微调已定型参数（掉电保存）；
> 位置环 PD 属板凳期定型，验收对象是波形形态，必须走 VOFA。

## 0. 一次完整调参会话（流程总览）

```text
┌ 准备（一次性）
│  ① 接线：VOFA 串口(UART5, PA1/PA0) + 视觉串口 + 步进总线(UART7)，云台机构可自由转动
│  ② 视觉上位机就绪：能回选题确认帧、对静态目标持续出坐标（§2）
│  ③ PC 开 VOFA+：JustFloat / 230400，建 13 通道 + 7 个命令控件（§2 两张表）
├ 进入条目（板上按键：K1=上 K2=下 K3=确认 K4=返回）
│  ④ 上电 → OLED 菜单一级列表 → 选 DEBUG(K3) → 选 GimbalTune(K3)
│     OLED 显示 RUNNING 横幅；VOFA 波形开始走（此时云台必然不动：安全初值 DB=10000）
│  ⑤ 看 CH6(state)：应 1→2→3；停在 1 超 1s 变 4 = 视觉没应答 → 查视觉后发 GO=1 重试
├ 调参（VOFA 发命令看波形，§3 六步 SOP，每步有判据）
│  ⑥ 第0步 测噪声定 DB 下限 → 第1步 极性首验（最小能量，最重要的安全步）
│     → 第2步 定 MS → 第3步 拉 kp → 第4步 加 kd → 第5步 收 DB
├ 收尾
│  ⑦ 抄下 VOFA 里的终值（XP/XD/YP/YD/DB/MS，以回显通道为准）
│  ⑧ K4 退出条目（确定性安全停，步进保持力矩）
│  ⑨ 回填 app_compose.c → s_gt_cfg.aim 四组字段 → 重新编译烧录（§3 第 6 步）
└  ⑩ 复验（可选但推荐）：烧录后重进条目，把回填的同一组值重新发一遍命令，
      做一次阶跃对比波形与调参时一致——确认抄写/回填没出错
```

要点：**退出条目/断电即丢 VOFA 会话里的值**（安全设计：重进一律重置安全初值），
第 ⑨ 步回填才是"调参完成"；中途坐标断流/握手超时 → state=4，发 `GO=1` 原地恢复，不丢已调值。

## 1. 控制律与参数含义（1 分钟版）

每收到一帧视觉坐标（≈10ms 一拍）：

```text
error = 坐标 − 图像中心           （像素，float）
delta = kp*error + kd*(error−上帧error)   （本拍脉冲增量）
      → |error| ≤ DB 则 delta=0（死区，防抖）
      → |delta| 限幅到 [1, MS]（floor-1 + 每拍步长封顶）
      → 累加进轴绝对目标 → 一帧 0xAA 发双轴步进
```

| 参数 | 命令 | 作用 | 调大了会怎样 | 调小了会怎样 |
|---|---|---|---|---|
| kp（X/Y 轴） | `XP`/`YP` | 误差→脉冲比例，收敛速度主旋钮 | 超调、来回振荡 | 收敛慢、爬行 |
| kd（X/Y 轴） | `XD`/`YD` | 阻尼，压超调 | 抖动被放大（D 放大噪声） | 超调压不住 |
| 死区 px | `DB` | 误差小于它就不动（止抖核心） | 瞄不准（稳态误差=DB） | 目标附近抖动 |
| 步长上限 | `MS` | 每拍最多走几脉冲（斜率限制），有效域 [1,10000] | 大误差时猛冲 | 大误差时收敛慢（匀速爬） |
| 重握手 | `GO` | 发 `GO=1` 重发选题握手（STOPPED 后恢复会话） | — | — |

**两条硬事实**（源自实现，调参时必须知道）：

1. **kp=0 不等于不动**。只要误差在死区外，最小也走 1 脉冲/帧（floor-1 设计，保证能爬进死区）。
   所以「安全不动」靠 DB=10000（进条目时的默认值），不靠零增益。**不要在 DB 很小时把增益全清零
   放着不管**——它会以 1 脉冲/帧匀速爬到轴程限位。
2. **无积分项，也不需要**。步进的目标累加本身就是积分器：静态目标下纯 P 就能零稳态误差
   （收敛到死区内）。不要指望加 I，没有这个参数。

## 2. 接线与上位机配置

- **VOFA 串口**：UART5（PA1/PA0），**230400 8N1**，协议 **JustFloat**。
- **视觉串口**：视觉上位机需按《视觉通信协议_主控确认版_v1》工作——
  能对 `0xFF 主 子 0xFE` 选题帧回确认帧，并持续输出坐标帧（静态目标持续出坐标）。
  条目进入时固件自动发选题 `(1,0)`（占位号，改 `app_compose.c` 的 `GIMBALTUNE_TOPIC_*`）。
- **VOFA 通道表**（JustFloat 13 通道，按序）：

| CH | 内容 | CH | 内容 |
|---|---|---|---|
| 0 | err_x（像素误差 X） | 7 | XP 回显 |
| 1 | err_y | 8 | XD 回显 |
| 2 | delta_x（本拍脉冲增量 X） | 9 | YP 回显 |
| 3 | delta_y | 10 | YD 回显 |
| 4 | cur_x（轴绝对目标脉冲 X） | 11 | DB 回显 |
| 5 | cur_y | 12 | MS 回显 |
| 6 | state（1=握手 2=ARMING 3=瞄准 4=停） | | |

- **VOFA 命令控件**：文本形如 `XP=0.2\n`（firewater 命令格式，命令名区分大小写）。
  建议做 7 个按钮/滑条：`XP` `XD` `YP` `YD` `DB` `MS` `GO`。
  回显通道显示的是**清洗后的生效值**（负数会被截成 0、MS 最小 1）——命令发没发成功看回显。

## 3. 调参 SOP（按顺序做，每步有判据）

进入条目后的初始状态：增益 0、DB=10000、MS=1 → **云台必然不动**；
state 走 1→2→3（若停在 1 超 1s 变 4：视觉没回确认，检查视觉后发 `GO=1` 重试）。

### 第 0 步 · 测噪声，定 DB 下限

什么都别动，把目标放在偏离中心处，看 CH0/CH1（err）波形的毛刺峰峰值。
**DB 终值 ≥ 噪声峰峰值的一半再留 50% 裕量**（例：噪声 ±3px → DB≥4.5，取 5~6）。
先不要下调 DB——本步只记数，DB 保持 10000。

### 第 1 步 · 极性首验（最重要的安全步）

极性接反 = 正反馈跑飞，必须最先验，且用最小能量验：

1. 只动 X：`MS=5`、`DB=<第0步终值>`、`XP=0.01`（YP 保持 0，Y 轴纹丝不动）。
2. 把目标放在中心偏右，看 CH0（err_x）：**幅值应该缓慢变小**（每拍最多 5 脉冲地爬向目标）。
3. 若 err_x 反而变大（云台向反方向转）→ **立即 BACK 退出条目**（确定性安全停），
   改 `app_compose.c` 里 `s_gt_cfg.aim.sign` 对应轴的符号（+1↔−1），重新编译烧录再来。
   **极性故意不做成 VOFA 命令**——防止运行中误发反转造成跑飞。
4. Y 轴同法（`XP=0`、`YP=0.01`）。

### 第 2 步 · 定 MS（步长上限）

MS 是每拍斜率限制，按步进速度算上限：

```text
每拍物理可走脉冲 ≈ step_speed_rpm / 60 × 每转脉冲数 × 0.01s
例：30 RPM、3200 脉冲/转 → 30/60×3200×0.01 = 16 脉冲/拍
```

**MS 取物理上限的 1~2 倍**（例中取 16~32）：太小→大误差时命令已饱和、波形是匀速斜线、
调 kp 没反应；太大→目标一跳命令猛冲。先取物理上限值。

### 第 3 步 · 拉 kp（单轴阶跃响应）

1. `XP` 从 0.05 起，每次 ×1.5~2。每改一次做一次**阶跃**：把目标快速挪开 50~100px，
   看 CH0 收敛形态 + CH2（delta_x）是否长时间顶着 ±MS（顶满=饱和，判据失真，回第 2 步加 MS）。
2. 目标形态：**err 单调或带一次小过冲地归零**。
   - 收敛慢、后段匀速爬 → kp 偏小（后段 |delta|<1 被 floor 顶着）；
   - 到位后过冲一大截再回来、甚至来回 2~3 次 → kp 偏大；
   - 持续等幅振荡 → 记下此值 kp_osc，**终值取 kp_osc 的 40~50%**。
3. 视觉链路有延迟（采集+处理+传输，几十 ms），延迟越大能用的 kp 越小——以波形为准，不背数字。

### 第 4 步 · 加 kd 压超调

kp 定型后若仍有过冲：`XD` 从 kp 的 2~5 倍起加（kd 乘的是每帧误差差分，量纲天然比 kp 大一档）。
判据：过冲变平、收敛拍数不变或更短。**加到 err 稳态段毛刺明显变大就退一半**——
D 项在放大视觉噪声（坐标已被上位机 Kalman 滤过，固件不再滤，D 的噪声上限就在这）。
Y 轴重复第 3、4 步（Y 轴负载/惯量不同，参数常与 X 不同）。

### 第 5 步 · 收死区（止抖）

把 DB 从第 0 步终值继续微调：目标静止时 CH2/CH3（delta）应**恒为 0**（不许 ±1 来回蹦）；
仍蹦 → DB+1 再看。DB 每加 1px，瞄准稳态误差就多 1px——够稳就停，不要多给。

### 第 6 步 · 回填终值

把定型的 XP/XD/YP/YD/DB/MS 抄进 `app_compose.c` → `s_gt_cfg.aim` 的
`kp/kd/deadband_px/max_step_pulse`（逐轴数组；DB/MS 若两轴要不同值，此处可分开填），
重新编译烧录。**退出条目/断电即丢 VOFA 里的值——回填才算完成。**

## 4. 异常速查

| 现象 | 原因 | 处置 |
|---|---|---|
| state 停在 1，1s 后变 4 | 视觉未回确认帧 | 查视觉侧，`GO=1` 重试（不丢已调参数） |
| state=3 但坐标一停 0.5s 就变 4 | 坐标流中断（目标丢失/串口断） | 恢复目标后 `GO=1` 重握手 |
| 云台朝反方向转、误差变大 | 极性反 | 立即 BACK 退出，改 `s_gt_cfg.aim.sign`（第 1 步） |
| 增益清零后云台仍缓慢单向爬 | floor-1 爬行（死区外最小走 1 脉冲） | 把 DB 拉大（如 10000）或退出条目 |
| 改 kp 波形没变化 | delta 顶满 MS 饱和 | 看 CH2/CH3 是否贴着 ±MS，加 MS |
| 到位后小幅高频抖 | DB 小于噪声 或 kd 过大 | 先加 DB（第 5 步），再降 kd |
| 命令发了回显不变 | 命令名/格式错（区分大小写，需 `\n` 结尾） | 核对第 2 节命令表 |
| BACK 退出后云台不垂落、带阻力 | 步进保持使能力矩（设计如此=安全态） | 正常，无需处理 |

## 5. 边界与安全（固件已内置，操作者知晓即可）

- 进条目必不动（DB=10000 安全初值）；BACK 退出=确定性停止下发（步进保持力矩）。
- 坐标断流 0.5s / 握手无应答 1s → 自动停（state=4），不会保持最后一条大命令。
- 轴程软限位 `travel_limit_pulse`（占位 ±800/±400 脉冲，按机械行程实测改 `s_gt_cfg`）兜底，
  但**第 1 步极性验证前不要依赖它**——先小能量验极性。
- 步进速度由 ARMING 期预设的 `step_speed_rpm`（默认 30，上限 100）决定，MS 只是每拍目标增量上限。

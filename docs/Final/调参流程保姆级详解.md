# 调参流程保姆级详解（没摸过车也能照做）

> 这份文件是 `OLED菜单操作手册.md` 五条调参流程的**逐步展开版**：每一步写到
> 「按哪个键 → 屏幕/电脑上会看到什么 → 这一步到底在干什么 → 记什么数 → 什么算好/坏」。
> **你现在手上没有硬件也没关系**——这就是拿到车那天照着做的剧本，每批末尾都写了
> 「现在没硬件能提前做什么」。数值出处以手册 §0.5 为准，本文只负责讲透。

---

## 0. 先把名词说成人话（遇到不认识的词回来查这里）

| 词 | 人话解释 |
|---|---|
| **标定** | 车对世界的认知是错的，你拿尺子/量角器量出真实值，把车脑子里的换算系数改对。例：车以为自己走了 1 米，实际走了 1.1 米——把换算系数改对，让它下次说 1 米就真走 1 米。**标定=对表，不是调性能。** |
| **增益（Kp/Ki/Kd）** | 纠错的"用力程度"。Kp=看到偏差立刻用多大劲拉回来；Ki=偏差一直不消就越来越用力（消"总是差一点"）；Kd=快速变化时提前收力（防止来回甩）。数字越大劲越大——太小=肉，太大=抖/甩。 |
| **PID / 速度环 / 外环** | PID=用上面三个劲儿自动纠错的算法。速度环=管"轮子转多快"的内层 PID；循迹环/位置环=骑在它上面的外层，外层只下指令"左轮 0.3m/s"，内层负责真转到 0.3。**内层不出力，外层说什么都白搭**——所以永远先调速度环。 |
| **milli 口径** | 屏幕显示的数 = 真实值 × 1000。屏上 `200` 就是 0.2，屏上 `100000` 就是 100。为什么：屏幕不好显示小数。**例外**：`Dist`（就是毫米）、`Ang`（就是度）、`S1/S2 deg`（就是度），这仨直读。 |
| **VOFA** | 电脑上的免费串口示波器软件（VOFA+）。车把数据用 USB 串口板发给电脑，你能看到实时波形；也能在它的命令框里给车发指令。只有调速度环必须用它，其他流程全靠板上按键。 |
| **阶跃** | 突然给一个目标（比如轮速从 0 直接命令到 0.3 m/s），看实际值多快追上去、会不会冲过头——这是看 PID 好坏最直观的办法。 |
| **参数组 / SAVE** | 板上菜单里的 TUNE/DRIVE/CHAS/HEAD/SERVO 就是五个参数组。改参数只改在内存里，**光标移到 SAVE 按 K3 才写进芯片 flash**（掉电不丢，开机自动加载）。任何一组按 SAVE=所有组一起存。SERVO 组例外，没有 SAVE。 |
| **架空** | 把车垫起来让轮子悬空。轮子随便转车也不跑，是最安全的试验姿势。 |
| **mm_per_pulse** | 编码器（轮轴上的转角计数器）每跳一个数，轮面前进多少毫米。车靠"脉冲数×这个系数"知道自己走了多远。**这个系数现在是拍脑袋的占位值**，所以第一次跑距离必然不准——批次 D 教你把它标准。 |
| **梯形剖面** | 定长直行的走法：先加速、中间匀速、快到了自动减速、停在目标点。速度曲线画出来是个梯形，所以叫这名。它没有 Kp/Ki/Kd，旋钮是 Cruise/Start/Accel/Decel/Dist 五个。 |
| **航向 / IMU** | 航向=车头朝向的角度。IMU=陀螺仪模块，串口报角度给主控。转弯和"走直不跑偏"都靠它。 |
| **heading_sign / 极性** | 硬件接线/安装方向造成的"正负号接反"。症状是方向反着来（该左转它右转）。**这类问题固件都留了唯一的改号开关，报给我改一个字重烧就好，永远不要用调参数去硬掰方向。** |

**按键就四个，所有流程通用**：
K1=上/加一步 K2=下/减一步 K3=进入/确认(在 SAVE 上=存盘) K4=返回/**急停**。
**跑车条目运行中屏幕锁定 RUNNING，只认 K4**——所以任何调参都是死循环：
**改参数 → 进条目跑 → K4 停 → 再改 → 再跑**，改和跑永远不同时。

**批次依赖顺序**（为什么是这个顺序）：

```
批A 上电自检 → 批B 速度环(地基,必须第一个) ─┬→ 批C 循迹      (还需: 灰度已在GrayTest标定)
                                             ├→ 批D 定长直行  (还需: 距离标定)
                                             └→ 批E 转弯航向  (还需: IMU已在ImuTest验过)
批F 舵机独立,随时可做
```

## 批次 A：开箱第一次上电（10 分钟，不跑车）

**目的**：确认板子、屏幕、按键、存档都活着，别的什么都不做。

1. 只接电池/电源，**先不接电机驱动的电机线也行**（更稳妥）。上电。
2. 屏幕应显示组列表：`DEBUG TUNE DRIVE CHAS HEAD SERVO`。
   - 黑屏 → OLED 线或供电问题，停,报修。
3. K1/K2 上下移动光标，K3 进 `CHAS` 组，看 `C Kp` 的数字：
   - **第一次上电它就是 0，正常**——速度环增益还没调过，车此时"有腿没劲"，
     这是故意的安全设计（没调参前车绝不会自己乱跑）。
4. K4 退回，进 `DEBUG`，随便选一个诊断页（比如 `BeaconTest`）K3 进、K4 出——
   进得去出得来、K4 一按就停，按键链路就算通过。
5. 之后每次上电都做第 3 步：**`C Kp` 不是 0 = 存档还在**；变 0 = 存档丢了，
   按你纸上的记录重新录入 + SAVE（所以下面每批都让你抄纸，就是防这一天）。

**没硬件现在能做**：把本文打印；电脑装好 VOFA+ 软件；备齐：USB 串口板、卷尺、
量角器、秒表、黑纸条、一支笔 + 下面附录的记录表打印一张。

## 批次 B：调速度环（唯一要连电脑的一批，约 30~60 分钟）

**目的（人话）**：教会两个轮子"让你转多快你就转多快"。调好之前，循迹/直行/转弯
全都不能开工。**完成标志：断电重启后，不连电脑，车轮也能精确执行速度命令。**

### B1. 接线和软件设置（第一次照抄，之后不用再设）

1. 车**架空**（轮子悬空能自由转）。
2. USB 串口板接电脑;杜邦线三根：主控 **PA1 → 串口板 RX**、**PA0 ← 串口板 TX**、**GND↔GND**。
3. 打开 VOFA+：左下角选对串口号 → 波特率 **230400**、8N1 → 协议引擎选 **JustFloat** → 打开串口。
4. 板上进 `DEBUG → SpeedTune`（K3）。此刻 VOFA 里应该出现 10 条通道的波形。
   - 没波形 → 串口号/波特率/协议三个设置挨个检查；再不行 PA0/PA1 两根线对调试试
     （TX/RX 视角是最常见坑）。

### B2. 认识屏幕上的 10 条线（只主看两条）

| 通道 | 是什么 | 用途 |
|---|---|---|
| ch6 / ch7 | 左/右轮**目标**速度 | 你命令它多快 |
| **ch8 / ch9** | 左/右轮**实测**速度 | **主看：它追目标追得怎么样** |
| ch0~ch5 | 左右轮当前 Kp/Ki/Kd | 确认命令生效了、抄数用 |

### B3. 发命令调参（VOFA 命令框输入，回车发送，立即生效）

1. 先给一组起手增益（量级是手册 §0.5 算过的，不会离谱）：
   依次发 `LP=150` `RP=150` `LI=100` `RI=100` `LD=0` `RD=0`。
2. 给阶跃：发 `LM=0.3` `RM=0.3`（两轮目标 0.3m/s）。轮子会转起来。
3. 盯 ch8 贴 ch6（左轮）、ch9 贴 ch7（右轮），按症状开药：

| 波形长什么样 | 说明什么 | 怎么改 |
|---|---|---|
| 实测慢吞吞爬向目标 | 劲不够 | Kp 加：`LP=200` 再看，一次加 30~50 |
| 到了目标附近却总差一小截平着 | 稳态差 | Ki 加：`LI=150`，一次加 30~50 |
| 冲过目标又跌回来、来回摆 | 劲过头 | Kp 减，或 Ki 减 |
| 高频毛刺乱抖 | 别加 D | 本车速度环**永远 LD=0 RD=0**（测速有量化噪声，D 只会放大它） |

4. 每改一次都重新给阶跃看效果：发 `LM=0` `RM=0` 停轮 → 再发 `LM=0.3` `RM=0.3`。
5. 合格线：阶跃后实测**约 0.2 秒内**贴住目标、不来回摆、平稳段不抖。左右轮分别看。
6. 调完：发 `LM=0` `RM=0` 停轮，**把最终 Kp/Ki 抄在纸上**（看 ch0~ch5 的数值确认）。

### B4. 存档（不做这步，断电全白调）

> 为什么：VOFA 里改的增益只活在内存。CHAS 组 + SAVE 才写进 flash，开机自动加载。

1. K4 退出 SpeedTune → 进 `CHAS` 组。
2. 屏显=真实×1000：假设你调出 Kp=180、Ki=120，那么 `C Kp` 要按到 `180000`、
   `C Ki` 按到 `120000`。步长一步 10000，从 0 起分别按 18 下、12 下 K1。`C Kd` 留 0。
3. 光标移到 `SAVE`，按 K3。屏幕存盘。
4. **验证**：整车断电 → 重新上电 → 进 `DEBUG → SpeedTune` → 看 ch0~ch5，
   显示的就是你存的值 = 存档链路全通。发 `LM=0.2` 轮子立刻响应 = 大功告成。

**没硬件现在能做**：把 B3 的症状表读熟；在记录纸上预填"起手 LP/RP=150、LI/RI=100"。

## 批次 C：调循迹（不用电脑，看车姿态）

**目的**：车能咬住黑线跑。**前提**：批 B 已存档；灰度已在 `GrayTest` 页用黑纸标定过
（电位器拧到黑白分明——那是接灰度硬件当天做的事，见杜邦验收清单）。

1. **先验方向（架空）**：进 `DEBUG → LineFollow`，手拿黑线（或把车架在线上）左右移动，
   看轮子差速方向：探头偏到线左边时，车轮的反应应该是"往右修让线回到中间"。
   - **方向反了**（越偏越使劲往外拐）：这是灰度左右位序问题，去 GrayTest 用黑纸碰
     **车头最左**探头、记下亮的是第几位、发我改一个映射值。**不要调参数硬掰。**
2. **第一次放地上跑，大概率直愣愣冲出去不拐弯——正常**：出厂 `LF Kp=0`，
   意思是"看见偏了也不打方向"。这不是坏了。
3. 进 `TUNE` 组，`LF Kp` 从 `10` 起（步长 2，从 0 按 5 下），SAVE，回 LineFollow 跑：

| 车的姿态 | 怎么改 |
|---|---|
| 见弯不拐、直道漂出线 | Kp 加（一次加 4~6，即按 2~3 下） |
| 直道走蛇形、左右画龙 | Kp 略减；或 `LF Kd` 从 0 加到 5~20 压抖 |
| 弯道外甩丢线 | Kp 加；还甩=速度太快（见下面注） |
| 丢线后 1.5 秒自动停车 | 这是保护不是故障：丢线找不回就停 |

4. 每次改完 → SAVE → 跑 → K4 停 → 再改。`LF Ki` 一般永远 0（弯道积分会兜圈）。
5. 合格线：目标赛道全程不丢线、直道不画龙。**终值抄纸上。**
> 注：循迹的行驶速度当前是代码里保守写死的 0.25 m/s，不在菜单里。稳了想提速找我改一行重烧。

**没硬件现在能做**：无。这批纯靠车和赛道。

## 批次 D：定长直行 + 距离标定（用户点名批——逐字展开）

**目的分两件事，别混**：
- **D-1 标定**：让车"以为走了 1 米"="真走了 1 米"（对表，一次性）；
- **D-2 调参**：让它走这段路又快又稳、停得准（性能，反复调）。

**前提**：批 B 已存档（速度环没劲车不动——进条目不动别慌，先查 `C Kp` 是不是 0）。

### D0. 先认识 DRIVE 组五个参数（都是干嘛的）

| 参数 | 人话 | 默认(屏显) | 什么时候动它 |
|---|---|---|---|
| `Dist mm` | 这次要走多远，毫米 | 1000（=1米） | 标定用 1000；比赛按题目要求设 |
| `Cruise` | 中段匀速多快 | 200（=0.2m/s） | 求快时加，一步 20 |
| `Start` | 起步瞬间给多快 | 80（=0.08m/s） | 起步卡住不动时加 |
| `Accel` | 提速多猛 | 300（=0.3m/s²） | 加速太肉时加 |
| `Decel` | 刹车多猛 | 300 | **冲过头时加（最常用的一个）** |

默认这套是"保守但能跑"的：进条目就会慢悠悠走约 1 米停下。

### D1. 先干什么：跑一次看看（熟悉流程）

1. 车放地上，前方留 ≥1.5 米空地。
2. `DEBUG → ProfiledStraight`，K3 进入 → 车加速-匀速-减速-自己停。
3. 任何时刻 K4=急停。车最多跑 15 秒强制停（防失控底线）。
4. 想再跑一次：K4 退出 → K3 再进（每进一次跑一段）。

### D2. 标定 mm_per_pulse（把"车的里程表"对准，只做这一次）

**这段是干嘛的，先说透**：车算距离靠数轮轴脉冲，"脉冲数 × mm_per_pulse = 走了多少毫米"。
这个系数取决于轮子直径和编码器规格，**现在固件里填的是占位猜测值**，所以车"以为的 1 米"
和真实的 1 米**必然对不上**——可能跑 80cm 就停，也可能跑到 1.2m。把系数改对的过程就叫标定。
这个系数在代码里，**你量数，我改值重烧**，来回一两趟就准。

一步步做：

1. 地上用胶带贴一条起点线，车尾（或前轮轴，选一个基准以后都用它）对齐起点。
2. `DRIVE` 组确认 `Dist mm = 1000`（默认就是）。
3. 跑一次 ProfiledStraight，等它自己停稳。
4. **卷尺量**：起点线到车的同一个基准点，量出实际走的距离，记作 **L**（毫米）。
5. 把 **L 发我**。我按公式 `新系数 = 旧系数 × L ÷ 1000` 改代码重烧
   （例：实跑 1100mm → 系数放大 1.1 倍；实跑 900mm → 缩小到 0.9 倍）。
6. 烧完复跑第 3~4 步验证。**合格线：命令 1000mm，实跑在 980~1020mm 之间（±20mm）。**
   没进就再报一轮 L，通常两轮内收敛。
7. 标定完成后这件事**永远不用再做**（除非换轮子/换编码器）。

### D3. 再调性能（对症下药，每改一项 SAVE 再跑）

| 现象 | 原因 | 动哪个 |
|---|---|---|
| 冲过目标才停 / 撞前面 | 刹车不够狠 | `Decel` 加（一步 50，加 2~4 步看） |
| 停得太猛、车头点头/打滑 | 刹车太狠 | `Decel` 减 |
| 起步卡一下才走 / 原地嗡嗡 | 起步速低于车的"最小能动速" | `Start` 加（一步 10） |
| 全程太肉太慢 | 敢快一点 | `Cruise` 加 + `Accel` 加 |
| 距离设短了跑不到设定速度 | 路太短爬不到巡航速就得刹车 | **正常现象**，不用调 |
| 快停时原地小转不往前 | 终段航向纠偏盖过前进速度 | 报我，减 hold 差速上限 |
| 起步后 0.8 秒被强制停 | 堵转保护触发（命令在动、轮没动） | 查轮子是否被卡 / `Start` 太低 |

调满意后 `SAVE`（Dist 和四个剖面参数一起存）。**终值抄纸上。**

### D4. 顺带的第二个标定：走直（可选，转弯批之后做更顺）

直行 1 米如果**侧偏超过 ±5cm**：说明两轮天生不完全一样劲，需要开航向保持——
去 `HEAD` 组把 `H Kp` 从 10 起加（见批 E ④），它管"直行时用陀螺仪把车头扶正"。

**没硬件现在能做**：贴起点线的胶带、卷尺放进工具箱；记录纸上预画 D2 的表格
（命令值 1000 / 实测 L / 第几轮）。

## 批次 E：转弯与航向（HEAD 组）

**目的**：K3 一按，原地精确转 90°（或你设的角度）。**前提**：批 B 已存档；
IMU 已在 `ImuTest` 验过健康（valid=1、静置漂移合格——接 IMU 硬件当天的事）。

1. `HEAD` 组先认参数：`T Kp`=转弯劲（本批主调）、`Ang`=每次测试转几度（默认 90，
   步长 15）、`H Kp/Ki/Kd`=直行扶正劲（批 D4 用，本批不动）。
2. 车放地上（原地转，不用跑道），在车正前方贴一条胶带作 0° 基准。
3. `T Kp` 起手 `20`（=0.02），SAVE。
4. `DEBUG → TurnTest`，K3 进 → 车原地转 Ang 度自动停（最长 10 秒强制停）。
5. **第一次进就转反了方向**（命令+90° 它往右/左转错边）：IMU 极性问题，
   停下来发我，改 heading_sign 一个字重烧。**别用参数硬掰。**
6. 量角器（或对地面胶带目测刻度）量实际转角，对症：

| 现象 | 怎么改 |
|---|---|
| 转不到位就停（差好几度） | `T Kp` 加（一步 5） |
| 冲过头再往回摆一下 | `T Kp` 减 |
| 到位前极慢地蹭最后几度 | 正常（越接近目标劲越小）；蹭太久 `T Kp` 略加 |
| 转到一半 10 秒被强制停 | `T Kp` 太小根本转不动 → 加 |

7. **合格线：命令 90°，实转 88°~92°（±2°）**，连转 4 次回到原朝向（累计误差看得见）。
8. 满意后 SAVE，终值抄纸。然后顺手做批 D4（直行扶正 `H Kp`）。

**没硬件现在能做**：无。

## 批次 F：舵机摆位（机构装好后随时 10 分钟）

**目的**：找到机构（夹爪/挡板…）的"开"和"合"两个角度值。

1. **空转态**（没跑任何条目）进 `SERVO` 组，`S1 deg` 设 90（中位，安全起点）。步长 5°。
2. `DEBUG → ServoTest` K3 进 → 舵机立刻转到 90° 并保持住劲。
3. K4 退出 → 舵机松劲（能用手拨动）。改角度再进，反复逼近机构到位的角度。
4. 找到"开=X°、合=Y°"后**把两个数写在纸上**——SERVO 组特意不存 flash
   （角度是机构装配的事实，赛题定型后我把终值写死进代码）。
5. 卡点：进页舵机抖/复位重启=舵机在从主控板偷电，**必须独立 5V 供电、只共地**；
   转到某角度嗡嗡响=顶到机械限位，记下角度发我设软限位。

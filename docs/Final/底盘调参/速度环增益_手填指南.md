# 速度环（内环轮速 PID）增益 — VOFA 调 + 手填指南

> 对应代码：`hc-team/app/service/chassis/chassis.c`（速度环）+ `hc-team/app/service/tuning/`（VOFA 调参）。
> 速度环是**内环**，循迹环 / 位置环都级联在它上面（外环给目标轮速 → 速度环给 PWM → 电机）。

---

## 1. 为什么要手填

- 速度环增益（轮速 PID 的 Kp/Ki/Kd）目前**只能用 VOFA 现场调**（`DEBUG/SpeedTune` + 上位机），**不入 flash、不掉电保存**。
- 开机默认增益是 **0**（`chassis.c` 的 `k_pid_cfg`），装配层不设——**不填就出不了力，循迹/位置环命令了也不动。**
- 所以流程是：**VOFA 调出好值 → 记下来 → 手填进代码 → 重编烧录**，这样增益就烧进固件、开机直接可用，不用每次连 VOFA。

> 循迹外环增益（TUNE 组）和位置环剖面（DRIVE 组）是**按钮调 + 存 flash** 的；只有速度环这一层还是手填。要把它也做成按钮持久化是另一件事（本指南不涉及）。

---

## 2. 填在哪（唯一位置）

**文件：`hc-team/app/service/chassis/chassis.c`，`k_pid_cfg` 结构（约第 27 行）。**

```c
/* 初始 PID 配置：增益 0（调参前不出力），限幅 = 电机口径，微分不过滤 */
static const Pid_Config_T k_pid_cfg = {
    .kp = 0.0f, .ki = 0.0f, .kd = 0.0f,   /* ← 把 VOFA 调好的三个值填这里 */
    .out_limit = (float)MOTOR_OUTPUT_MAX, /* 别动：输出限幅 = 电机口径 ±1000 */
    .integral_limit = 0.0f,               /* 别动：≤0 → 按 out_limit 推导 */
    .d_filter_alpha = 1.0f,               /* 别动：微分不过滤 */
};
```

- 这是**共享参数接口**：`Chassis_Init` 把这**同一份**应用到**左右两轮**（`chassis.c` 里循环两侧 `Pid_Init`）。
- **只改 `.kp / .ki / .kd` 三个 `0.0f`**，其余三项（out_limit / integral_limit / d_filter_alpha）**不要动**——它们是电机口径/限幅约定。
- 改完 `rtk make -C Debug all` 重编烧录。开机 `Chassis_Init` 即应用，位置环/循迹环直接出力。

### 左右轮要不同增益怎么办？

`k_pid_cfg` 是**两轮共用一份**。若 VOFA 里左右调出的值差不多（多数小车如此），填一份即可。
若左右差异大、必须分开，则不能只改 `k_pid_cfg`——需在装配层（`sys_init.c` 的 `Chassis_Init()` 之后）显式补两句：

```c
Chassis_SetSpeedGains(CHASSIS_SIDE_LEFT,  kpL, kiL, kdL);
Chassis_SetSpeedGains(CHASSIS_SIDE_RIGHT, kpR, kiR, kdR);
```

`Chassis_SetSpeedGains(side, kp, ki, kd)`（`chassis.h`）是**单侧运行时设增益接口**，即时生效、限幅归 PID cfg，可从正常控制流调用。

## 3. 操作流程（VOFA 调 → 手填）

1. **架空轮子 / 低速**，进 `DEBUG/SpeedTune`，连 VOFA 上位机。
2. 用 VOFA 调左右轮速度环 Kp/Ki/Kd，看目标 vs 反馈曲线跟随（tx 十通道：kp/ki/kd L、kp/ki/kd R、目标 L/R、反馈 L/R）。
3. 调到跟随好、不抖不超调，**把这几个值记下来**。
4. 退出 SpeedTune，打开 `chassis.c`，把值填进 `k_pid_cfg`（或左右分开则填装配层两句）。
5. `rtk make -C Debug all` 重编烧录。
6. 复位，直接进 `DEBUG/ProfiledStraight` 或 LineFollow 验证——现在不连 VOFA 也出力了。

---

## 4. 坑与注意

- ⚠ **填完别再进 SpeedTune**：进 SpeedTune 会用它自己的 cmd（从 0 起）**覆盖**速度环增益，把你烧进去的值冲掉（直到你在 VOFA 里再调上去）。要验证烧进去的值，直接跑 ProfiledStraight/LineFollow，别进 SpeedTune。
- ⚠ **out_limit / integral_limit 不要动**：限幅唯一所有者是 PID cfg（±1000 PWM = 电机口径），乱改会影响所有用到速度环的地方。
- ⚠ **堵转保护还在**：即便速度环增益偏大，起步卡住时位置环的**堵转看门狗**（~0.8s）会切停（见位置环手册 §8）——但主防线仍是别把增益/起步速配到堵转。
- 速度环增益**不掉电保存**是当前已知局限；手填进 `k_pid_cfg` 相当于"用重编烧录代替 flash 持久化"，值就固化在固件里了。

---

## 5. 一句话

> **VOFA/SpeedTune 调好速度环 → 记下 Kp/Ki/Kd → 填进 `chassis.c` 的 `k_pid_cfg`（两轮共享）→ 重编烧录 → 开机直接可用，别再进 SpeedTune 冲掉它。**

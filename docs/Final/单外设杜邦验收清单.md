# 单外设杜邦验收清单（打杂间隙 10 分钟一项）

> 2026-07-23 生成。前提：主控板已烧当前 main 分支固件。每项独立，拿到哪个外设就测哪个。
> 通用流程：接线（先查 `docs/Final/硬件对接确认清单.md` 第三节确认引脚）→ 上电 → 按键进
> DEBUG 组对应条目 → 对照「看什么/合格线」 → BACK 退出（所有条目退出都是确定性安全停）。
> 需要 VOFA 的项：VOFA 接 UART5（PA1=TX→USB 串口 RX、PA0=RX、GND 共地，230400 8N1，JustFloat）。

## 0. 主控板裸板（其他一切的前提）

| 项 | 内容 |
|---|---|
| 接线 | 只接电源 + OLED/按键模块（PA29/PA30 I2C + PB4/PB5/PB25/PA14） |
| 验收 | 上电 OLED 出菜单；K 键能进出 DEBUG/TUNE/DRIVE/CHAS/HEAD/SERVO 六组 |
| 不亮 | **先怀疑 H1**（硬件表 PA29=SDA/PA30=SCL 与固件相反）：对调两根线再试；哪种接法亮，回报我定案 syscfg |

## 1. 蜂鸣器 + LED —— `BeaconTest`

| 项 | 内容 |
|---|---|
| 接线 | 蜂鸣器 PB18（H4 冲突未决——若板上蜂鸣器无效，杜邦独立蜂鸣器到 PB18）；LED 板载 PB22 |
| 条目 | DEBUG → BeaconTest（七种模式各 2s 轮播：单响/双响/长响/常亮/慢闪/快闪/声光同步） |
| 合格线 | 人耳人眼逐个对上节拍；退出即静默 |
| 异常 | 一直响不停=蜂鸣器无源或低电平触发（回报我换 PWM/反极性，接口不变） |

## 2. IMU —— `ImuTest`（需 VOFA）

| 项 | 内容 |
|---|---|
| 接线 | IMU 模块 ↔ PA25（主控 RX）/PA26（主控 TX）+ 3.3V/GND |
| 条目 | DEBUG → ImuTest；VOFA 8 通道：ch0=yaw ch1=角速度 ch2=age(数据年龄) ch3=valid ch4=**drift(0偏误差)** ch5=帧计数 **<u>ch6=校验错 ch7=溢出</u>**(0) |
| 合格线 | ch3=1；**ch5 每秒 ≈+1000**（每周期 2 种帧 × 本项目配置 500Hz）；ch6/ch7 恒 0；**静置 ≥60s 后 ch4 漂移 \|drift\|≤0.05°/s**（器件 Kalman 模块，超了先查供电与震动再怀疑器件） |
| ch5 异常判读 | 只 +20/s = 模块速率还在**出厂 10Hz**（波特率倒是配对了）→ 重跑上位机配置；ch6 狂涨或 ch3=0 = 模块波特率还在**出厂 115200**（固件收 230400）→ 整套配置没做，按 `IMU陀螺仪配置指南.md` 末尾一句话流程走一遍（改 230400 + 500Hz + 零偏校准 + 存盘，存进模块自身、断电不丢） |
| 手转测试 | 手转模块 90°，ch0 变化 ≈90（差太多=波特率/量程问题，回报） |

## 3. 电机 + TB6612 —— `MotorDir` → `SpeedTune`

| 项 | 内容 |
|---|---|
| 接线 | TB6612 模块杜邦到 H2 扩展口：PWM PA22(左)/PB2(右)，方向 PA13/PA12(A)、PA17/PA24(B)；**电机电源独立供，只共地** |
| 条目1 | MotorDir：两轮 ±200 前后 2s 循环——记录每轮实际转向（这一步定「谁是左轮/正方向」，错了回报我改宏，硬件不动线） |
| 条目2 | SpeedTune（需 VOFA + 编码器已接）：悬空调 PI，看阶跃跟随 |
| 合格线 | 方向循环准确换向、退出即停；SpeedTune 阶跃无持续振荡 |

## 4. 编码器 —— `EncoderTest`（需 VOFA）

| 项 | 内容 |
|---|---|
| 接线 | 左编码器 A/B → PB10/PB11（U33），右 → PB7/PB9（U34），5V/GND |
| 条目 | EncoderTest：ch0/1=累计脉冲 L/R，ch2/3=速度 L/R |
| 合格线 | **手朝车头前进方向拨轮，两侧脉冲都应变正**；变负=AB 相反（预期内事故，回报我改 `s_direction_sign` 唯一修正点，别动线） |

## 5. 灰度 12 路 —— `GrayTest`（OLED 自绘面板，可不接 VOFA）

| 项 | 内容 |
|---|---|
| 接线 | NCHD1 12 路 → PB24/PB0/PB14/PB21/PB19/PB17/PB26/PB20/PB8/PB13/PB12/PB27（=硬件 G1..G12），**5V 供电+串阻方案不变** |
| 条目 | GrayTest：OLED 四行面板（实时/粘滞/跳变/十六进制） |
| 合格线 | 黑纸逐个探头划过，对应位逐个翻转；无常亮/常灭死位 |
| **必做记录** | 用黑纸只触发**车头最左**探头，记下 OLED 上是哪一位亮——这是 bit0 左右定案（Q12）的实测输入，告诉我 |
| 注意 | 阈值在硬件电位器不在固件；G8(PB20) 若表现异常见确认清单 H3 |

## 6. 舵机 ×2 —— `ServoTest` + SERVO 组

| 项 | 内容 |
|---|---|
| 接线 | 舵机1 信号 PA27、舵机2 PB1；**舵机 5V 独立供电，只共地**（堵转电流会拉复位 MCU） |
| 流程 | ①空转态 SERVO 组把 S1 设 90（安全中位）→ ②进 ServoTest：舵机应摆到 90 并保持 → ③BACK（释放，可手拨）→ ④改 S1=45 → 再进 → 摆到 45 |
| 合格线 | 到位无持续抖动；0/180 端点不顶死机械（顶死就记下实测限位角，告诉我设软限位） |

## 7. ESP32-C3 无线 —— `LinkTest`（需 VOFA；**引脚未定案前只验固件在位**）

| 项 | 内容 |
|---|---|
| 现状 | 端口占位：进条目 ch5=port_absent 恒 1、ch0=alive 恒 0 —— **这是如实显示，不是 bug** |
| 引脚定案后 | ①我改 syscfg 两行 → ②先不接 ESP32：把该 UART 的 TX-RX 杜邦短接（自环回）→ alive=1 即字节层验收 → ③接 ESP32-C3 双机：两台车各自 alive=1、ch1 收帧数持续涨 |
| 合格线 | 自环回 alive=1；双机 ch2 校验错不涨 |

## 8. 视觉（香橙派）—— `VisionLink`（需 VOFA；没有视觉硬件时用电脑代打）

| 项 | 内容 |
|---|---|
| 接线 | 视觉 UART：PA8/PA9 ↔ 香橙派（**H2 问题：若收不到，把两根线对调一次定案**），230400 8N1 |
| 无视觉硬件 | 电脑 USB 串口杜邦接上来，用串口工具发帧代打（帧格式见下），一样能全绿 |
| 帧速查 | 坐标：`AA 55 09 01 x(f32小端) y(f32小端) CRC16-MODBUS(小端,范围=len+payload)`；状态：`AA 55 03 02 b0 b1 CRC`；选题握手（主控→视觉）：`FF 主 子 FE`，视觉原样回显即确认 |
| 合格线 | ch6=confirmed 变 1（回显握手后）；打坐标流时 ch2=coord_seq 匀速涨、乱码/断帧后能自恢复；ch7=retries 停止增长 |
| **要向视觉组要的** | 状态帧 2 字节位域的语义表（识别结果编码）——拿到发我，5 分钟接进消费端 |

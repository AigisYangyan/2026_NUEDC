# 赛前 48 小时极限调参任务表（上电就调）

> 2026-07-23 生成。设计前提：**整车只在赛前两天到手**。所有参数走板载按钮组
> （TUNE/DRIVE/CHAS/HEAD/SERVO）+ K3 SAVE 掉电保存——**调好一步存一步，
> 断电重来不清零**。VOFA 只用来看波形，不再是参数的家。
> 纪律（diansai-skills §4/§5）：先测量后结论；全过就收手，不追求指标好看；
> 每步带「放弃线」——超时就锁上一档参数往下走，宁可慢车跑完，不要快车翻车。

## 阶段 B：整车到手日（T-48h ~ T-24h）——自底向上，每步 SAVE

**顺序不可换**：方向→测速→速度环→灰度→循迹→距离标定→航向。上层全靠下层。

| # | 任务（条目/参数组） | 做法 | 合格线 | 时间预算 | 放弃线 |
|---|---|---|---|---|---|
| B1 | 方向自检（MotorDir） | 架空看四种组合 | 前后向与命令一致 | 10min | 无（必须过，改宏 5min） |
| B2 | 编码器极性（EncoderTest） | 手推车前进 | 双侧脉冲同正 | 10min | 无（必须过） |
| B3 | **速度环**（SpeedTune+VOFA，悬空） | 阶跃 0.3m/s 看跟随；PI 不加 D（速度环范式）；抗饱和在位 | 上升无持续振荡、稳态贴给定 | 60min | 90min：取「不振荡的最大 Kp 的 60%」锁定 |
| B4 | **录入持久化**：CHAS 组填 B3 终值 → SAVE | 按钮录 milli 值 | 断电重启→SpeedTune 复测行为一致 | 10min | 无 |
| B5 | 灰度标定（GrayTest） | 赛道实布光线下电位器逐路调 | 黑白翻转干脆、无死位 | 20min | 换备用传感器 |
| B6 | **循迹**（LineFollow + TUNE 组，落地低速） | 先 0.25m/s 直道→弯道；只调外环 Kp/Kd（Ki 最后微量）；速度在 DRIVE 组 | 直道不蛇形、标准弯不丢线、**连跑 3 圈行为不衰减** | 90min | 120min：降 straight_speed 一档锁参 |
| B7 | **距离标定**（ProfiledStraight + DRIVE 组） | 命 1000mm 实跑，卷尺量实测 L：新 mm_per_pulse = 旧值 × 1000/L（告诉我或按公式改配置重烧）；复测 | **停点误差 ≤±20mm**（送货停线验收数） | 40min | 60min：±30mm 收手，靠循迹段兜 |
| B8 | **航向/转弯**（TurnTest + HEAD 组） | 先 IMU 极性（转错向→heading_sign）；T Kp 从 5(milli) 起加到 90° 干脆到位；H Kp 同法（ProfiledStraight 开 heading_hold 验直线） | **90°±2°**（量角器）；直行 1m 侧偏 ≤±50mm | 60min | 90min：T Kp 取「不过冲最大值的 70%」锁参 |
| B9 | 全量 SAVE + **参数抄纸**（CHAS/TUNE/DRIVE/HEAD 全部值） | flash 已存，纸是双保险 | 断电重启全部行为复现 | 10min | 无 |

**B3 速度环出问题速查**：抖动=Kp 大或测速噪声（先降 Kp）；跟不上=Ki 太小；
换向顿挫=正常（motor.c 换向死区在管，别调它）。

| C4 | 声光/舵机/无线的赛题动作点插入段表 | Alert_Play/Servo/Link 都是现成 API | 该响的响、该亮的亮 | 发挥项可弃 |
| C5 | 终版 SAVE + 抄纸 + 烧录备份 .out | 双车都烧同版 | — | — |

**C3 满载不过决策树**（一次只动一个，diansai-skills §5）：
```
满载不过 → ① DRIVE 组 Accel/Decel 降一档（最便宜，通常就够）
          → ② CHAS 组 Kp 适度上调（Ki 不动）
          → ③ 仍罩不住 → 叫我立第二组参数档（载荷开关自动切换）
```

## 出问题找谁

- 现象对不上任何合格线 → 先按「单外设清单」回退到该外设单测，分清硬件/固件；
- 固件要改的（宏/极性/段表/mm_per_pulse）→ 发我现象+数字，改点全是唯一所有者，都是分钟级；
- **不要现场手改代码里的增益**——一切增益走按钮组，改代码=丢掉 SAVE 体系。

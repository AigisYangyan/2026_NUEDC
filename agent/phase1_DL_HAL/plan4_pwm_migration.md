## Design Answers Before Coding

1. Abstraction: the motor driver owns the public duty command contract, not a generic PWM service.
2. Hidden details: timer instances, capture/compare indexes, PWM period assumptions, and DriverLib calls stay inside `motor.c` or a tiny board-specific runtime helper.
3. Module boundary: `motor.h` should expose motor concepts only. It must not expose `HC_HAL_PWM_Channel_e`, Timer register types, SysConfig macro names, or DriverLib details.

## Scope

Allowed implementation work:

- `hc-team/driver/motor/motor.h`
- `hc-team/driver/motor/motor.c`
- `hc-team/driver/mspm0_runtime/mspm0_runtime.h`
- `hc-team/driver/mspm0_runtime/mspm0_runtime.c`, only if a tiny board-specific PWM helper is preferred over direct private helpers in `motor.c`
- `hc-team/app/system/sys_init.c`, only to verify PWM counters are started once and to remove duplicate startup if Plan4 makes it redundant

Out of scope:

- Do not migrate non-motor PWM users unless they are proven to exist inside `hc-team`.
- Do not edit `project/mspm0/board.syscfg` or generated SysConfig files.
- Do not remove `sdk/hal` globally.
- Do not migrate UART, I2C, OLED, MPU6050, AT24Cxx, or common `HC_*` aliases.
- Do not add a generic PWM HAL replacement.

## Required Mapping

Use these SysConfig PWM symbols:

| Motor side | PWM instance | Capture compare index | Existing SysConfig source |
| --- | --- | --- | --- |
| Left | `PWM_MOTOR_L_INST` | `GPIO_PWM_MOTOR_L_C1_IDX` | TIMA0 CC1 on PA22 |
| Right | `PWM_MOTOR_R_INST` | `GPIO_PWM_MOTOR_R_C0_IDX` | TIMA1 CC0 on PB2 |

The compare write standard is:

```c
DL_TimerA_setCaptureCompareValue(PWM_MOTOR_L_INST, duty, GPIO_PWM_MOTOR_L_C1_IDX);
DL_TimerA_setCaptureCompareValue(PWM_MOTOR_R_INST, duty, GPIO_PWM_MOTOR_R_C0_IDX);
```

Use the exact SysConfig symbols generated by the active project. If generated names differ, stop and update this plan before coding.

## Public Contract

Preserve the current public motor behavior:

- `Motor_SetPwm(Motor_T *m, float pwm)` accepts `-1000.0f` to `1000.0f`.
- Values above `1000.0f` clamp to `1000.0f`.
- Values below `-1000.0f` clamp to `-1000.0f`.
- The PWM compare value is the absolute duty from `0` to `1000`.
- Positive and negative signs only select direction pins. Compare value is always non-negative.
- Zero command drives the motor direction state used by the existing implementation for zero and writes compare `0`.
- `Motor_Brake()` drives both H-bridge direction pins high and writes compare `0`.

## Motor Data Structure Standard

- Remove `#include "hc_hal_pwm.h"` from `motor.h` and `motor.c`.
- Remove `HC_HAL_PWM_Channel_e pwm_ch` from `Motor_T`.
- If the driver needs to identify left/right inside a helper, use `Motor_Id_e` or a private enum, not an exported PWM channel type.
- Keep SysConfig details private to `motor.c` unless a runtime helper is chosen.
- Prefer a private helper shape like this:

```c
static void motor_write_duty(Motor_Id_e id, uint16_t duty)
{
    if (id == MOTOR_LEFT) {
        DL_TimerA_setCaptureCompareValue(PWM_MOTOR_L_INST, duty, GPIO_PWM_MOTOR_L_C1_IDX);
    } else if (id == MOTOR_RIGHT) {
        DL_TimerA_setCaptureCompareValue(PWM_MOTOR_R_INST, duty, GPIO_PWM_MOTOR_R_C0_IDX);
    }
}
```

This is an interface shape standard, not a requirement to copy exact code. The final implementation must match local style and compile cleanly.

## Timer Startup Standard

- PWM counters must be started once during system startup, not inside every `Motor_SetPwm()` call.
- Plan2 currently provides `Mspm0Runtime_StartMotorPwm()` from `SysInit()`.
- Plan4 must verify there is no duplicate repeated startup in the motor driver.
- At startup, both motor compare values must be zero before or at the same time counters become active.
- If `Mspm0Runtime_StartMotorPwm()` already writes zero compare and starts both counters, keep that as the single startup owner.

## Acceptance Criteria

Plan4 is done only when all of these are true:

- No `hc_hal_pwm.h` include remains in `hc-team`.
- No `HC_HAL_PWM_*` name remains in `hc-team`.
- No `HC_HAL_PWM_CH_*` enum remains in motor structures or initialization.
- `Motor_SetPwm()` preserves the `-1000.0f` to `1000.0f` public input contract and clamps out-of-range values.
- `Motor_SetPwm()` writes direction pins and direct PWM compare values using SysConfig symbols and DriverLib, or through a local board-specific helper with the same direct implementation.
- `Motor_Brake()` drives both H-bridge direction pins high and writes compare `0`.
- PWM counters are started exactly once during system initialization.
- No generic PWM HAL replacement was introduced.
- The Debug build succeeds.

## Blockers

Stop and record a blocker instead of guessing if:

- `PWM_MOTOR_L_INST`, `PWM_MOTOR_R_INST`, or the capture compare index macros are missing or renamed.
- PWM period is not `1000` counts and the old duty scale cannot be mapped directly.
- Direction pin migration from Plan3 is incomplete and motor safety cannot be preserved.
- Direct compare writes produce inverted duty behavior compared with the old HAL.
- Startup no longer guarantees zero compare before PWM counters run.
